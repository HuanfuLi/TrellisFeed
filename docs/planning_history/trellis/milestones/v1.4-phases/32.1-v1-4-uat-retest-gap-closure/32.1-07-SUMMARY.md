## Outcome

Incremental classification pipeline now produces clean concept-noun anchor names instead of paraphrased questions. Operator-reported case ("What is spaced repetition and why does it work?" creating anchor "Spaced repetition and why does it work" instead of "Spaced Repetition") will no longer reproduce on newly-classified questions.

## Root cause

`buildStepPrompt` at `canonical-knowledge.service.ts:477` returned the same generic prompt template for all three pipeline steps (branch / cluster / anchor). The anchor step had no naming guidance — model freely returned a question paraphrase.

The fallback `classifyAndAnchor` prompt at line 857-876 already had the correct guidance ("1-3 word concept noun phrase, NEVER a sentence", with the literal "Spaced Repetition" example). But that path only runs when the incremental pipeline fails, so the fix never reached production.

## Fix

`buildStepPrompt` adds a `namingHint` string when `level === 'anchor'`, appended to both the no-existing-anchors branch and the select-or-create branch. Branch/cluster prompts unchanged.

10 lines added, 2 modified. tsc clean (0 errors).

## Decisions

- **Hint placement**: appended at the end of the candidate list, before the JSON-format instruction. Keeps it close to the `name` field the LLM is filling in.
- **Branch/cluster left alone**: operator's reported issue is anchor-only; branch/cluster names look correct in current data.
- **No post-LLM validation guard**: if the LLM still ignores guidance, bad names land in storage. Deferred — easier to add a guard if the prompt change proves insufficient than to undo guard logic later.
- **Existing data not migrated**: anchors created before this commit keep their wrong names. Operator's existing "Spaced repetition and why does it work" anchor will still show in badges until manually renamed or until a migration is built. Out of scope — minimal change.
