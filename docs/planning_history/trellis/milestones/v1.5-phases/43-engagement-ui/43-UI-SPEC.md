> Visual + interaction contract for wiring the Phase 39 engagement service into the masonry feed, `/saved` archive, and PostDetailScreen deep-dive flow. Generated by gsd-ui-researcher.

This contract pre-populates from CONTEXT.md (LP-01..LP-05 / SV-01..SV-04 / DD-01..DD-05 / TS-01 / DS-01) and RESEARCH.md. All values are concrete — no exploratory ranges. Reference notation `{file:line}` points to the existing pattern source.

| Role | Value | Phase 43 Usage |
|------|-------|----------------|

| Secondary (30%) | `var(--surface-variant)` (#F5F0E8 / #1F1F1F) | Deep-dive button background, sheet row hover/press, list-card thumbnail fallback |
| Accent (10%) | `var(--primary-40)` (#558B2F / #4CAF50) | Active tab indicator, active segmented-control segment text, deep-dive button label, "Restore standard" link, filled `Bookmark` corner icon |
| Saved corner icon | `var(--primary-40)` filled | LP-03 saved-state visual signal |
| Liked corner icon | `var(--node-salmon)` (#FFAB91 / #1E2326) filled | LP-03 liked-state visual signal; semantically warm + distinct from primary green |
| Destructive | `var(--danger)` (#E53935 / #EF5350) | **Not used for "Not interested" row** (operator preference: subtle, not destructive — see Notes below) |
| Muted text | `var(--muted-foreground)` (#6B6B6B / #A0A0A0) | Inactive tab text, "Not interested" row icon + label, deep-dive icon, header bookmark icon (resting state) |
| Border / divider | `var(--border)` (rgba(0,0,0,0.08) / rgba(255,255,255,0.10)) | Tab-bar bottom border, segmented-control border, list-card divider (none — cards float on shadow) |

**Accent reserved for (explicit list):**
1. Active tab underline + text (`SavedScreen` tab bar)
2. Active segmented-control segment background + white text (deep-dive Standard/Deep toggle)
3. Deep-dive CTA button label text
4. "Restore standard" link text during deep stream
5. Filled `Bookmark` corner icon on saved tiles
6. Header bookmark icon in **active** state (when `location.pathname === '/saved'` is open as Outlet — but since `/saved` unmounts HomeScreen visually, the icon's resting + hover states are sufficient; no active-state tinting needed)

**Color rules (do not violate):**
- The "Not interested" dismiss row is **NOT** tinted red/destructive. Use `var(--muted-foreground)` for icon + label. Rationale: dismiss is a non-destructive UX (data persists via `engagementService.dismissedAnchorIds`; reversible via Force-New-Day or future undismiss surface). Operator framing of dismiss is conservational, not punitive.
- The liked corner icon uses `--node-salmon`, NOT `--danger`. A "heart" must not read as an error/destructive state.
- The header bookmark icon at rest is `var(--muted-foreground)`, not accent — matches CLAUDE.md `feedback_tile_simplicity_preference.md` (one signal per surface; the icon's *job* is navigation, not state).

**Row state-flip rule (LP-04):**

The menu, on open, calls `engagementService.isSaved(postId)` and `engagementService.isLiked(postId)` synchronously. Each call resolves to a boolean; the row's label, icon-fill, and click handler dispatch the inverse action:

| Save row state | Label | Icon | onClick |
|----------------|-------|------|---------|
| `!isSaved` | `t('engagement.menu.save')` = "Save" | `<Bookmark fill="none">` | `engagementService.savePost(postId)` |
| `isSaved` | `t('engagement.menu.unsave')` = "Unsave" | `<Bookmark fill="currentColor">` | `engagementService.removeSavedPost(postId)` |

| Like row state | Label | Icon | onClick |
|----------------|-------|------|---------|
| `!isLiked` | `t('engagement.menu.like')` = "Like" | `<Heart fill="none">` | `engagementService.likePost(postId)` |
| `isLiked` | `t('engagement.menu.unlike')` = "Unlike" | `<Heart fill="currentColor" color="var(--node-salmon)">` | `engagementService.unlikePost(postId)` |

**Dismiss row (always one state):**

| Label | Icon | onClick |
|-------|------|---------|
| `t('engagement.menu.dismiss')` = "Not interested" | `<EyeOff fill="none">` | `engagementService.dismissAnchor(anchorId)` followed by sheet close + toast + HomeScreen fade-out (see §4) |

**Animation:**

- No framer-motion inside the sheet — relies on `BottomSheet`'s existing CSS transitions. Adding framer-motion would not propagate `<MotionConfig reducedMotion="user">` through the React-portal boundary, so prefer CSS.
- `prefers-reduced-motion` already respected at the CSS level (the BottomSheet's `transition` is a single property; OS-level reduce-motion settings will not strip it but the 300ms duration is acceptable).

**Tap-outside / ESC:**

- Tap on backdrop → `BottomSheet onClose` (existing).
- ESC key → handled by BottomSheet's portal-level keydown listener (verify; if not present, add a window keydown listener in LongPressMenu that calls `onClose` on `Escape` — optional, mobile-first).
- Drag-down dismiss: **not implemented this phase**. CONTEXT.md LP-01 mentions it as a desirable property; RESEARCH §2 explicitly defers it. Backdrop tap is sufficient.

**Anti-wire invariant:**

- LongPressMenu MUST NOT emit `CONCEPT_EXPLORED`. Only emits via the engagement service (which fires `ANCHOR_DISMISSED` for dismiss and `ENGAGEMENT_CHANGED { kind, id }` for save/like/unsave/unlike per Phase 39 D-05).
- Source-reading test enforces `grep 'CONCEPT_EXPLORED' app/src/components/LongPressMenu.tsx` returns 0 matches.

### 4. Dismiss fade animation (LP-05)

**Mechanic:**

When the dismiss row in LongPressMenu commits, HomeScreen's `ANCHOR_DISMISSED` subscriber fires and filters `dailyPosts` in-place:

```typescript
setDailyPosts(prev => prev.filter(p => p.sourceQuestionIds?.[0] !== anchorId));
```

The visual fade is provided by wrapping each tile in `MasonryFeed.tsx` with `<AnimatePresence>` (one AnimatePresence per column at `MasonryFeed.tsx:455-457` rendering columns).

| Property | Value |
|----------|-------|
| Wrapper | `<AnimatePresence initial={false}>` around each column's tile list (column A + column B) |
| Tile container | The existing `<motion.div>` at `MasonryFeed.tsx:421-440` (already animated for entrance) gains `exit={{ opacity: 0, scale: 0.96 }}` |
| Non-animated tile container | The else-branch `<div>` at `MasonryFeed.tsx:442-451` becomes `<motion.div>` to participate in AnimatePresence exit. Keep `variants` undefined so it does not re-enter on layout shifts; only the `exit` prop matters. |
| Exit transition | `{ duration: 0.2, ease: [0.25, 0.1, 0.25, 1] }` — matches Phase 42 tile motion vocabulary |
| Stagger | **none** — all same-anchor tiles fade simultaneously (operator framing: "one-frame consistency") |
| Layout reflow | After AnimatePresence exit completes, masonry recomputes column heights (existing `useEffect([items])` at `MasonryFeed.tsx` already handles this) |
| Reduced motion | `<MotionConfig reducedMotion="user">` already wraps MasonryFeed (Phase 42 D-03). When reduced motion is on, AnimatePresence collapses exits to instant — acceptable. |

**Edge case:** if AnimatePresence wrapping adds complexity beyond scope, the executor MAY ship instant removal in v1 (per RESEARCH "Risks" mitigation row: "Can simplify to instant removal for v1"). The CONTEXT.md LP-05 specifies 200ms fade — preferred — but executor has fallback authority if AnimatePresence integration touches more than the column tile lists.
