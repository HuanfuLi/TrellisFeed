## Critical Gap Re-Audit

### CONTENT-03: Queued News Multi-Source Grounding

**Previous finding:** The first v1.5 audit found direct news generation was correct, but queued-news prefetch cached only `filtered[0]`, so normal queued news posts could still produce one-source `newsMeta.sources`.

**Current finding:** Closed by Phase 46.

Evidence:

- `app/src/services/concept-feed.service.ts` defines `PreFetchCache.news` as `Map<string, WebSearchResult[]>`.
- The refillQueue prefetch loop calls `selectNewsTopSources(results.data.results, usedDomains)` and stores `preFetched.news.set(a.conceptId, topSources)`.
- The cached branch uses `result = cached[0]` and `topSources = cached.slice(0, 3)`.
- `newsMeta.sources` is built with `mapNewsSourcesToNewsMeta(topSources)`, which assigns stable 1-based indexes.
- `app/src/services/post-essay.service.ts` still consumes `sources.slice(0, 3)` with no prompt rewrite.
- `cd app && node --test tests/services/concept-feed-source-diversity-wiring.test.mjs` passes 17/17 and verifies multiple sources plus stable indexes.
- The integration checker also ran the paired CONTENT-03 suites with 23/23 passing.

**Decision:** CONTENT-03 is satisfied. The prior audit blocker is closed.
