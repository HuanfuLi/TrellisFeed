<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user input → collection name → JSX | Collection name rendered in picker (custom collection rows), in toast labels, and in drill-in header (plan 50-08). React text-node escaping is the default mitigation. |
| Fuse match indices → JSX | Indices are integers from Fuse.js output; spliced as text via slice() before being wrapped in JSX. No HTML injection vector. |
| Concurrent picker toggles → service writes | Picker captures membership on open; commits diff on Done. Concurrent toggles before Done are draft-state only. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-50-XSS-NAME | Tampering / XSS | Collection name rendered in CollectionPickerSheet rows + toast | mitigate | Source assertion: no `dangerouslySetInnerHTML` in CollectionPickerSheet.tsx. Collection name comes from validated input (50-03 trim + length cap + dedup). React escapes children text by default. |
| T-50-XSS-HL | Tampering / XSS | Fuse match indices spliced into JSX via HighlightedText | mitigate | Source assertion: no `dangerouslySetInnerHTML` in HighlightedText.tsx. Text slices wrapped in `<mark>` JSX (React-escaped). Indices come from Fuse's internal pattern matcher operating on the same text — never user-controlled HTML strings. |
| T-50-PICKER-RACE | Race / Lost Update | Concurrent membership toggles in picker before commit | mitigate | Picker captures originalSaved + originalCollectionIds ONCE on open. Toggles update draft state only. Commit-on-Done computes diff and applies mutations. Test asserts no `engagementService.savePost` or `collectionService.addPost` call inside checkbox onClick handlers (only in handleDone). |
</threat_model>
