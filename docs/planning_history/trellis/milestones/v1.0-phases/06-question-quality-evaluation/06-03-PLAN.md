# Phase 6 Plan 03: Pattern Library Gap Closure

**Objective:** Expand PATTERN_LIBRARY to cover all common off-topic question types, making the filter reliable without LLM fallback.

## Tasks

### Task 1: Expand Meta-Question Pattern
**Goal:** Detect questions about the AI system itself (name, capabilities, how it works).

**Current pattern (line 19):**
```typescript
{ pattern: /^(what can you do|who are you|how do you work|tell me about yourself|what is echolearn|describe yourself|what are your capabilities)/i, confidence: 0.95 },
```

**Issue:** Misses variations like:
- "What's your name?" (possessive contraction)
- "What are your capabilities?" (already included)
- "How do you work?" (already included)
- "Are you serious?" (sarcastic meta)
- "Can you help?" (overly broad but common)

**Plan:**
Update pattern to:
```typescript
{ pattern: /^(what (can|should|will) you|who (are|is) you|how (do|does|can) (you|this) work|tell me (about|who|what) (you|yourself)|what (is|are) (your|the) (name|capabilities|purpose)|describe yourself|are you|can you (actually )?help)/i, confidence: 0.95 },
```

This uses word boundaries with alternation to catch:
- "what are your capabilities" ✓
- "what's your name" ✓
- "how do you work" ✓
- "are you serious" ✓
- "can you help" ✓

**File:** `app/src/services/question-filter.service.ts` line 19
**Confidence:** Keep at 0.95 (high confidence — these are rarely learning questions)

### Task 2: Expand Trivial Acknowledgment Pattern
**Goal:** Catch single-word acknowledgments and backchannel responses.

**Current pattern (line 28):**
```typescript
{ pattern: /^(ok|okay|yeah|yes|no|sure|thanks|thank you|np|yep|nope)$/i, confidence: 0.8 },
```

**Issue:** Uses exact match (^ ... $), so it fails if user types:
- "Alright" (not in list)
- "Got it" (not in list)
- "I see" (not in list)
- "Cool" (not in list)
- "Right" (not in list)

Also misses simple word boundary variants like adding punctuation (e.g., "ok.").

**Plan:**
Change exact match to word boundary match:
```typescript
{ pattern: /\b(ok|okay|alright|got it|i see|cool|right|yeah|yes|no|sure|thanks|thank you|np|yep|nope|uh huh|uh uh|mhm|mm|exactly|indeed|fine|whatever|absolutely|certainly|certainly|definitely|sounds good|for sure|totally|absolutely|100)\b/i, confidence: 0.8 },
```

This will match:
- "Alright" ✓
- "Got it" ✓
- "I see" ✓
- "Cool" ✓
- "ok." ✓ (word boundary ignores punctuation)

**File:** `app/src/services/question-filter.service.ts` line 28
**Confidence:** Keep at 0.8 (medium confidence — user might say "Alright, tell me about X" which is substantive)

## Rationale

**Why expand patterns instead of fixing LLM?**
- LLM endpoint error is likely a configuration issue outside code control
- Expanded pattern library provides reliable detection without LLM dependency
- Pattern matching is fast (<1ms), deterministic, and doesn't require external API calls
- Phase 6 design supports this: "Pattern-first with LLM fallback for ambiguous cases"
- With good patterns, LLM fallback becomes truly optional (for edge cases only)

**Why not use word boundaries everywhere?**
- Some patterns like "tell me a joke" need exact phrase matching to avoid false positives
- Others like "ok" can be word-boundary safe
- Trade-off: broader matching vs. false positives (resolved by confidence scores)
