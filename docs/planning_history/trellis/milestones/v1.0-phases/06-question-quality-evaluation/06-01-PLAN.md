<objective>
Implement intelligent filtering and routing of user questions to determine whether they should be stored in the canonical knowledge graph or kept only in chat session history.

**Phase goal:** Add a hybrid pattern + LLM-based detection layer that flags off-topic and meta-questions, allows users to see and override the flag with a minimal UI, and respects the flag during knowledge graph ingestion.

**Current behavior:** All questions are immediately persisted to the knowledge graph.

**Target behavior:** 
- Valid questions auto-save (current behavior unchanged)
- Off-topic questions are flagged with a small badge below the AI response
- Users can click the badge to see why it was flagged and choose to "save anyway"
- Only non-flagged questions (or overridden flagged questions) are ingested into the knowledge graph
- Flagged questions that aren't overridden stay in chat history only

**Why this matters:** Users can now curate their knowledge graph, preventing trivial or context-dependent chat messages from polluting their long-term learning resource.
</objective>

<follow_up_context_handling>
## **NEW: Session Context for Follow-Up Questions**

This section addresses how follow-up questions (clarifications, elaborations, "Can you explain more?") are evaluated.

### **Problem Statement**
The initial filtering logic evaluates each question in isolation. This creates a false-positive risk:
- User asks: "What is spaced repetition?" → Valid question, saved ✓
- User asks: "Can you elaborate?" → Pattern matches "very short trivial" or ambiguous question, gets flagged ❌

Without context, follow-up questions that are clearly elaborations become falsely flagged and create friction.

### **Solution: Pass Session Context to Filter**

**Design decision (LOCKED):**
- When a question is asked, pass the **immediate prior Q&A pair** as context to the filter function
- The filter evaluates: "Is this valid in the context of what we just discussed?"
- Follow-ups that clearly reference or elaborate on the prior question pass through as valid
- Standalone trivial questions (with no prior context) are still flagged

**Implementation:**
1. Modify `evaluateQuestion()` signature to accept optional session context:
   ```typescript
   export async function evaluateQuestion(
     question: Question, 
     sessionContext?: { 
       priorQuestion?: string; 
       priorAnswer?: string 
     }
   ): Promise<Question>
   ```

2. In the **LLM fallback**, inject session context into the prompt:
   ```typescript
   async function isOffTopicByLLM(
     content: string,
     sessionContext: { priorQuestion?: string; priorAnswer?: string } | undefined,
     llmConfig: LLMConfig
   ): Promise<boolean> {
     const contextInstructions = sessionContext?.priorQuestion 
       ? `Prior question: "${sessionContext.priorQuestion}"\nPrior answer preview: "${sessionContext.priorAnswer?.slice(0, 200)}..."\n\n`
       : '';
     
     const response = await chatCompletion(
       [
         {
           role: 'system',
           content: `${contextInstructions}You are a question classifier. Determine if the following is a genuine learning/knowledge question or off-topic chat. If this appears to be a follow-up or elaboration on the prior question, treat it as valid. Respond with ONLY "yes" if off-topic, "no" if genuine learning question.`
         },
         { role: 'user', content }
       ],
       llmConfig
     );
     // ... rest of logic
   }
   ```

3. In `question.service.ts`, **extract prior Q&A and pass to filter**:
   - When `questionService.ask()` is called, pass the current session's last user Q&A pair to the filter
   - This data is available in AskScreen context (session.messages)

4. In `AskScreen.tsx`, **pass session context when calling ask()**:
   - Extend the `ask()` signature to optionally receive `sessionContext`
   - Extract prior message pair from `session.messages` before calling `ask()`
   - Pass as: `questionService.ask(content, { sessionContext })`

### **Behavior Examples**

| Scenario | Filter Input | Context | Result |
|----------|--------------|---------|--------|
| "What is spaced repetition?" | pattern only | none | Not flagged ✓ |
| "Can you elaborate?" | pattern: short + trivial | Prior: "What is spaced repetition?" | NOT flagged (context override) ✓ |
| "lol" | pattern: trivial | Prior: "What is spaced repetition?" | Still flagged (trivial in any context) ❌ |
| "Tell me a joke" | pattern: entertainment | none | Flagged ❌ |
| "What does that mean?" | pattern: ambiguous | Prior: None/empty session | Sent to LLM → likely flagged ❌ |
| "What does that mean?" | pattern: ambiguous | Prior: "What is the capital of France?" | Sent to LLM with context → NOT flagged ✓ |

### **Out of Scope (Phase 7+)**
- Multi-turn conversation chains (e.g., remembering the topic across 5+ exchanges)
- Adaptive context window sizes (currently just prior Q&A pair)
- User-configurable context passing behavior
- Analytics on follow-up vs first-question flagging rates

    Create a new service that detects off-topic and meta-questions using a hybrid approach per locked decision D-01.
    
    1. **Pattern-based filtering (fast, deterministic):**
       Define a library of regex patterns that match obvious meta/off-topic questions:
       
       ```typescript
       const PATTERN_LIBRARY = [
         // Greetings and casual openers
         { pattern: /^(hello|hi|hey|good morning|good afternoon|good evening|greetings|hey there)/i, confidence: 0.9 },
         
         // Meta-questions about the system
         { pattern: /^(what can you do|who are you|how do you work|tell me about yourself|what is echolearn|describe yourself|what are your capabilities)/i, confidence: 0.95 },
         
         // Requests for jokes, entertainment, non-learning content
         { pattern: /^(tell me a joke|make me laugh|give me a riddle|say something funny|tell me a story|write a poem)/i, confidence: 0.95 },
         
         // Incomplete test messages
         { pattern: /^(test|asdf|xyz|lol|haha|lmao|xd|:/i, confidence: 0.85 },
         
         // Very short trivial responses
         { pattern: /^(ok|okay|yeah|yes|no|sure|thanks|thank you|np|yep|nope)$/i, confidence: 0.8 },
       ];
       ```
       
       Export function: `function isOffTopicByPattern(content: string): { flagged: boolean; confidence: number }`
       
       Logic:
       - For each pattern in PATTERN_LIBRARY, test if it matches the content
       - Sum up the confidence scores of all matching patterns
       - Return { flagged: totalConfidence >= 0.7, confidence: totalConfidence }
       
    2. **LLM fallback (for edge cases):**
       If pattern confidence is < 0.7 and > 0 (some patterns matched weakly), call LLM:
       
       ```typescript
       async function isOffTopicByLLM(
          content: string,
          sessionContext: QuestionFilterContext | undefined,
          llmConfig: LLMConfig
        ): Promise<boolean> {
         try {
           const response = await chatCompletion(
             [
               {
                 role: 'system',
                 content: 'You are a question classifier. Determine if the following is a genuine learning/knowledge question or off-topic chat (greetings, jokes, meta-questions, etc). Respond with ONLY "yes" if off-topic, "no" if genuine learning question.'
               },
               { role: 'user', content }
             ],
             llmConfig
           );
           const trimmed = response.trim().toLowerCase();
           return trimmed.startsWith('yes');
         } catch (err) {
           // Graceful degradation: if LLM fails, assume not off-topic
           console.warn('[EchoLearn] filter LLM fallback failed, assuming valid question:', err);
           return false;
         }
       }
       ```
    
    3. **Main exported function:**
       ```typescript
       export interface QuestionFilterContext {
          priorQuestion?: string;  // The immediately preceding user question
          priorAnswer?: string;    // The AI response to the prior question
        }
        
        export async function evaluateQuestion(
          question: Question,
          sessionContext?: QuestionFilterContext
        ): Promise<Question> {
         // Step 1: Check patterns
         const patternResult = isOffTopicByPattern(question.content);
         
         // Step 2: High confidence? Use pattern result.
         if (patternResult.confidence >= 0.75) {
           return { ...question, flagged: patternResult.flagged };
         }
         
         // Step 3: Low confidence? Try LLM fallback if available.
         if (patternResult.confidence > 0) {
           const settings = mockSettingsService.getSync();
           if (settings.llm.isConfigured) {
             const llmResult = await isOffTopicByLLM(question.content, sessionContext, settings.llm);
             return { ...question, flagged: llmResult };
           }
         }
         
         // Step 4: No patterns matched and no LLM available, assume valid.
         return { ...question, flagged: false };
       }
       ```
    
    Implementation notes:
    - Pattern library is a const array for easy iterative refinement
    - Use mockSettingsService (same pattern as question.service.ts, line 161)
    - LLM fallback is optional/graceful — if it fails, assume not off-topic
    - Keep pattern evaluation lightweight (no recursive calls)
    - Add console.warn for debugging (visible in browser DevTools)
