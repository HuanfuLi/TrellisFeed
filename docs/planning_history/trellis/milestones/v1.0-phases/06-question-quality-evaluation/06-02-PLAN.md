<objective>
Close gap identified in VERIFICATION.md: The non-streaming `question.service.ts` `ask()` method does not pass session context to `filterQuestion()`, causing follow-up questions evaluated via this path to be evaluated without prior Q&A context.

**Gap:** Line 259 of `question.service.ts` calls `filterQuestion(question)` with no second argument. The `ask()` method signature (line 161) only accepts `content: string`, so the caller cannot supply prior Q&A context.

**Impact:** Follow-up questions evaluated via the non-streaming path may be incorrectly flagged as off-topic. The streaming path (`useQuestions.askStreaming`) correctly passes context.

**Solution:** 
1. Extend `ask()` signature to accept optional `sessionContext` parameter
2. Pass `sessionContext` to `filterQuestion()` call
3. Document usage from AskScreen (whether this path is primary or legacy)

**Why this matters:** Ensures consistent behavior across all code paths. Users asking follow-ups via any path (streaming or non-streaming) will have accurate context-aware filtering.
</objective>
