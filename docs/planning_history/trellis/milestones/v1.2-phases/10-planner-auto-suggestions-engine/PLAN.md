# Phase 10: Planner Auto-Suggestions Engine

**Milestone:** v1.1 (Engagement & Discovery Iteration)  
**Status:** In Progress (Gaps Identified)
**Depends on:** v1.0 Planner foundation

## Goal

Implement automatic generation of Planner "Suggested Moves" when the Knowledge Graph is sufficiently populated and the Planner is empty. Support daily auto-refresh and trajectory-aware recommendation scoring. **Consolidate multiple 'Suggested Moves' sections into a single, unified UX.**

## Requirements

- **PLANNER-01**: Trigger auto-generation (5+ knowledge nodes AND empty Planner)
- **PLANNER-02**: Display suggestions automatically on Planner screen
- **PLANNER-03**: Daily auto-refresh (after podcast time)
- **PLANNER-05**: Trajectory-aware scoring algorithm
- **PLANNER-06 (UAT FIX)**: Consolidate redundant "Suggested Moves" sections (Auto-Suggested vs. Manual-Suggested)
- **PLANNER-07 (UAT FIX)**: Global accessibility for "Refresh" and "Skip All" buttons in the unified section

## Success Criteria

1. ✅ System detects 5+ nodes in Knowledge Graph AND empty Planner
2. ✅ Auto-generated suggestions appear on Planner screen without manual trigger
3. ✅ Suggestions are "Moves" linking to Posts, Questions, or Review sessions
4. ✅ Daily refresh triggers after podcast time (configurable)
5. ✅ Trajectory algorithm considers: review performance, question frequency, time since last review
6. ✅ Suggestions ranked by relevance score (0-100)
7. ✅ Suggestions persist across app restarts
8. ✅ No duplicate suggestions within 24 hours
9. ❌ Unified "Suggested Moves" section (no redundancy)
10. ❌ "Refresh" and "Skip All" buttons remain accessible regardless of section content

## Technical Approach

### Trajectory Data Model

```typescript
interface TrajectorySignal {
  reviewPerformance: number;        // 0-100, avg correctness on flashcards
  questionFrequency: number;        // questions asked in last 7 days
  timeSinceLastReview: number;      // milliseconds
  feedEngagement: number;           // posts viewed in last 7 days
  conceptCoverage: number;          // % of knowledge graph reviewed
  weakAreas: string[];              // concept IDs with low review scores
}

interface PlannedMove {
  id: string;
  concept: ConceptNode;
  moveType: 'review' | 'deepdive' | 'connection' | 'podcast';
  relevanceScore: number;           // 0-100
  reason: string;                   // "Time to review" or "Related to recent Q&A"
  targetTime?: number;              // ms (estimated time to complete)
  linkedResource?: {
    type: 'post' | 'question' | 'review';
    id: string;
  };
}
```

### Scoring Algorithm

```
relevanceScore = (
  reviewPerf_weight * (100 - reviewPerformance) +     // 0.4 - prioritize struggles
  timeSinceReview_weight * (timeSinceLastReview / maxTime) +  // 0.3 - prioritize overdue
  engagement_weight * feedEngagement +                 // 0.2 - surface popular
  coverage_weight * conceptCoverage                    // 0.1 - fill gaps
)

where:
  reviewPerf_weight = 0.4 (highest)
  timeSinceReview_weight = 0.3
  engagement_weight = 0.2
  coverage_weight = 0.1
```

## Task Breakdown

### Wave 1: Foundation & Data (Days 1-2)

#### T10.1: Trajectory Signal Aggregation
- Create `src/services/trajectoryAnalyzer.service.ts`
- Implement `aggregateSignals(): TrajectorySignal`
- Collect data from:
  - Review metadata (performance, timestamp)
  - Question history (count, timestamps)
  - Feed engagement (views, timestamps)
  - Knowledge graph (node count, coverage)
- Cache signals (update every 6 hours)
- **Acceptance:** Signals calculated correctly, reflects user activity

#### T10.2: Suggestion Scoring Engine
- Create `src/services/suggestionScorer.service.ts`
- Implement `scoreMove(move: ConceptNode, signals: TrajectorySignal): number`
- Apply weighting formula
- Rank suggestions by score descending
- Return top 5-8 suggestions
- **Acceptance:** Scoring works, results make sense

#### T10.3: Move Generation Logic
- Create `src/services/moveGenerator.service.ts`
- Implement `generateMoves(concepts: ConceptNode[], signals: TrajectorySignal): PlannedMove[]`
- Create different move types:
  - Review moves (link to existing flashcards)
  - Deep-dive moves (link to related posts)
  - Connection moves (link related concepts)
  - Podcast moves (AI-generated podcast on weak area)
- Map each suggestion to linked resource
- **Acceptance:** Moves generated for all concepts

#### T10.4: Auto-Generation Trigger
