<objective>
**Phase 10: Planner Auto-Suggestions Engine**

Auto-generate "Suggested Moves" in the Planner when users accumulate knowledge (5+ questions) but have empty planned activities. Suggestions refresh daily via event-driven (podcast completion) or time-based triggers. Uses deterministic trajectory scoring (not AI) to prioritize weak areas and create actionable learning moves.

**Purpose:**
- Prevent Planner stagnation when users have learned enough to need guidance
- Reduce friction: actionable suggestions appear without user initiative
- Encourage spaced repetition by suggesting overdue concepts
- Support cold-start users with equalized, diverse suggestions

**Output:**
- 4 core services (trajectory analyzer, scorer, move generator, auto-gen orchestrator)
- Settings UI consolidating podcast time + toggle
- Integration with existing planner, event system, and graph service
- Full test coverage (unit + integration tests for scoring, triggers, persistence)

**Confidence:** HIGH (all patterns sourced from existing services; no new dependencies)

- Settings UI + integration (Wave 4): ~4-5 hours
- Testing + validation (Wave 5): ~3-4 hours
- **Total: ~20-23 hours**

**Risk Level:** LOW
- Deterministic algorithm (no ML/randomness)
- 24h cooldown prevents regression loops
- Cold-start equalization handles new users
- Existing services provide all required signals
