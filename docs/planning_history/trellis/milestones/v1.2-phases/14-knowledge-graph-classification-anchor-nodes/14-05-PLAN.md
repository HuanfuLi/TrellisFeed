# Phase 14 Gap Closure Plan (Anchor Summary Fix)

## Goal
Fix the issue where Anchor Nodes are not receiving the intended <=80 word summary of each attached Q&A. The design specifies that the LLM generates a short summary during the classification call, which should be appended to the anchor, but the code currently falls back to a 200-character truncation of the raw answer.

## Root Cause
In `canonical-knowledge.service.ts`, the `classifyAndAnchor` method executes a secondary LLM call with a prompt that successfully requests a `"briefAnswer":"<=30 word answer for self-context"`. The result is parsed correctly into the `briefAnswer` property. However, on line 532, the code completely ignores this result and instead falls back to `question.shortSummary` (which is often undefined during streaming), `question.summary`, or an ugly slice of the raw answer:
```typescript
    const shortSummary = question.shortSummary || question.summary || question.answer.slice(0, 200);
    const summaryEntry = `[${question.id}] ${shortSummary.slice(0, 200)}`;
```

## Proposed Changes

### `app/src/services/canonical-knowledge.service.ts`
- **[MODIFY]** `classifyAndAnchor`
  - Update the `shortSummary` assignment around line 532 to prioritize the freshly generated `result.briefAnswer` before falling back to the existing question heuristics.
  - The new logic should look like this:
    ```typescript
    const shortSummary = result.briefAnswer || question.shortSummary || question.summary || question.answer.slice(0, 200);
    const summaryEntry = `[${question.id}] ${shortSummary.slice(0, 200)}`;
    ```
