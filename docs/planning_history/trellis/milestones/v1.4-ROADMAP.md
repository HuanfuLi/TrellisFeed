## Decisions of note

- **Test runner architecture:** Split `npm test` into `test:main` + `test:actions`. `test:actions` registers `_actions-mock-loader.mjs` via `--import` so trellis-actions tests can stub heavy LLM/SQLite/i18n deps. Cannot register globally because that breaks `web-search.test.mjs` (needs real settings.service).
- **Leaf-module pattern for testability:** Phase 36 extracted `feed-spread.ts` (spreadByConcept/spreadByStyle) and `refill-mutex.ts` (createPromiseMutex) so tests can exercise pure logic without importing services that drag in the i18n chain. Pattern documented in CLAUDE.md.
- **i18n leaf-module refactor deferred to v1.5:** 10 remaining test failures all transitively import `src/locales/index.ts:37` which uses Vite-only `import.meta.env.DEV`. Architectural cleanup; carried to v1.5 first wave.
- **Always-mounted screen state-resync principle:** Phase 36-14 documented that `useState(() => svc.get())` initializers fire ONCE at app boot for SwipeTabContainer slots. Screens reading services whose state changes mid-session MUST add `[location.pathname]` effects to re-sync. Documented in CLAUDE.md adjacent to Concept Feed Pipeline.
- **Force-New-Day handler symmetric two-cache mutation:** Phase 36-15 corrected Plan 36-13's "redundant" framing — `trellis_post_queue.date` and `trellis_daily_posts.date` are SYMMETRIC date-rejection caches, not redundant; both must be tripped under dev affordance because `today()` doesn't advance.
