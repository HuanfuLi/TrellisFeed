    ```bash

    ```
    For each hardcoded `toast('English message')`:
    - If in a screen (already handled by Plan 05), leave it — Plan 05's executor should have caught these
    - If in a component (handled here), replace with `toast(t('<namespace>.<key>'))` — requires the component to have `useTranslation()` wired
    - If in a service/hook/non-React file where `useTranslation()` is unavailable, use the imperative API:
      ```typescript
      import i18n from '../locales';
      toast(i18n.t('<namespace>.<key>'));
      ```
      This works from any module (services, event handlers, non-React code). `i18n.t` is a bound function that uses the current language.

    For toast strings shared across services (e.g. "Failed to connect"), use `common.toast.*` namespace.

    Step 4: Audit `throw new Error('English message')` where the message is user-facing (shown in UI).
