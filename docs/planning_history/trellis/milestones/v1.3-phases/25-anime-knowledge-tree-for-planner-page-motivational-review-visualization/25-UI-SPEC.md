## Color

EchoLearn uses a Material You Nature & Growth palette. The Trellis hero's color contract maps directly onto existing tokens вҖ” no new hex values introduced.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--surface)` `#FFFBF5` light / `#111111` dark | Hero card background fallback (visible behind asset while loading); tooltip surface |
| Secondary (30%) | `var(--surface-variant)` `#F5F0E8` light / `#1F1F1F` dark; trellis-bg asset itself | Background asset (Variants A/V); SVG wood/lattice fill (Variant C) |
| Accent (10%) | `var(--primary-40)` `#558B2F` light / `#4CAF50` dark | Primary CTA button; healthy leaf vein highlight; bloom glow on all-green state |
| Destructive | `var(--danger)` `#E53935` light / `#EF5350` dark | NOT USED in Phase 25 (no destructive flows on the hero вҖ” anchor deletion handled elsewhere) |

**Accent reserved for:**
1. Empty-state CTA button (`Ask a question вҶ’`) primary background
2. Tooltip primary action button (`Review`) primary background
3. All-green-state subtle bloom glow (D-43) вҖ” `box-shadow` rgba derived from `--primary-40`
4. Mount-animation vine stroke color when newly drawn (D-46) вҖ” base vine color uses branch-specific node tokens; the *animation* highlight uses `--primary-40` briefly
5. Bud вҶ’ leaf grow animation glow (D-42) вҖ” sparkle dots tinted with `--primary-40`

**Accent NOT used for:** body text, tooltip secondary button, leaf default fill, blossom fill, fruit fill, fallen-leaf decorative graphics.

### Lifecycle State Color Contract (per anchor leaf)

Locked by CONTEXT D-05/D-06; values map to existing `--node-*` tokens for visual continuity with GraphScreen.

| State | Visual Token | Hex (light) | Hex (dark) | Trigger |
|-------|--------------|-------------|------------|---------|
| Bud рҹҢұ | `var(--primary-80)` | `#BDDA9F` | `#2E7D32` | New anchor (`reviewCount === 0`) |
| Leaf вҖ” Green (healthy) | `var(--node-mint)` | `#B2DFDB` | `#1B2B1C` | All children `daysOverdue < 0` |
| Leaf вҖ” Yellow (warning) | `var(--secondary-40)` | `#FFD54F` | `#FFD54F` (preserved in dark) | Any child `1вҖ“7` days overdue |
| Leaf вҖ” Falling (orange) | `var(--node-peach)` | `#FFCCBC` | `#232320` | Any child `7вҖ“14` days overdue |
| Leaf вҖ” Fallen (brown/withered) | `var(--node-salmon)` desaturated 30% via `filter: saturate(0.7)` | `#FFAB91` | `#1E2326` | Any child `14+` days overdue OR aggregate `easeFactor < 1.5` |
| Blossom рҹҢё | `var(--accent-lavender)` | `#CE93D8` | `#78909C` | All children reviewed AND aggregate `easeFactor > 2.5` |
| Fruit рҹҚҺ | `var(--accent-coral)` | `#FF8A80` | `#4CAF50` | Blossom state for 7+ consecutive days |

**Vine color (per-branch, D-12):** Each branch uses one of `--node-mint`, `--node-sky`, `--node-lilac`, `--node-peach`, `--node-salmon` (deterministic by `hash(branch.id) % 5`). Stroke width: `2px`. Opacity: `0.85` so leaves visually pop.

**Fallen-leaf decorations (D-37, D-40):** Use leaf "Fallen" hex above; opacity `0.6`; positioned at hero baseline (bottom 8% of canvas).

**Glow / sparkle (D-41, D-43, D-45):** `box-shadow` or SVG `feGaussianBlur` filter using `color-mix(in srgb, var(--primary-40) 40%, transparent)`. Matches existing `glow-pulse` keyframe pattern in `app/src/index.css:381`.

## Interaction Contract

| Interaction | Trigger | Response | Animation |
|-------------|---------|----------|-----------|
| Tap leaf | `pointerdown` on leaf hit-target (44Г—44px) | Open tooltip popover anchored to leaf; close any other open tooltip first | Tooltip fade+scale-in: `opacity 0вҶ’1, scale 0.92вҶ’1`, 180ms `ease-out` (framer-motion `initial`/`animate`) |
| Tap outside tooltip | `pointerdown` outside tooltip bounds and outside originating leaf | Close tooltip | Fade-out 120ms |
| Tap Г— on tooltip | `click` on close button | Close tooltip | Same as above |
| Tap another leaf | While tooltip open | Close current tooltip, open new one | Sequential: close 120ms вҶ’ open 180ms (no overlap) |
| Tap `Review` in tooltip | `click` | `navigate('/review', { state: { anchorReview: { anchorId, anchorName } } })` вҖ” reuses Phase 15 `anchorReview` filter pattern from ReviewScreen | Default route transition |
| Tap `View Q&As` in tooltip | `click` | `navigate('/anchor/{anchorId}')` (existing AnchorDetailScreen route) | Default route transition |
| Tap empty-state CTA | `click` | `navigate('/ask')` | Default route transition |
| Scroll the page | natural scroll | Hero scrolls away with content (D-23, NOT sticky) | Native scroll |
| Leaf state change (REVIEW_COMPLETED) | eventBus event for anchor in current view | Recompute leaf state; if color changed, animate transition | 2s `ease` color tween + scale pulse 1вҶ’1.2вҶ’1 over 600ms + sparkle particles 800ms (D-38, D-41) |
| New anchor created (CLASSIFICATION_COMPLETED) | eventBus event | Insert bud at vine endpoint, grow into leaf | Scale 0вҶ’1 spring physics ~600ms, opacity 0вҶ’1 (D-42) |
| Anchor deleted (ANCHOR_DELETED) | eventBus event | Leaf fades + detaches (slight downward drift) | Opacity 1вҶ’0 + translateY(20px) over 800ms |
| Yellow вҶ’ falling transition | computed state change | Leaf detaches and falls | translateY downward + sinusoidal X drift + slow rotate, 3вҖ“4s (D-39) вҶ’ settle as fallen leaf at base (D-40) |
| All-green state achieved | computed | Tree-wide bloom glow + ambient sway | Subtle glow pulse on all leaves (`glow-pulse` keyframe analog) + low-amplitude sway loop (D-43) |
| Blossom unlock | state crosses to blossom | Leaf вҶ’ blossom petal-unfurl | Custom petal-unfurl 800ms (D-44) |
| Fruit appears | state crosses to fruit | Blossom вҶ’ fruit scale-in + glow | Scale 0вҶ’1 + glow fade 600ms (D-45) |
| Hero mount | first render | Vines draw on, then leaves pop in | SVG `stroke-dashoffset` draw per-branch staggered 200ms apart, leaves stagger 50ms (D-46) |
| Variant picker tap (dev only) | `click` on pill | Cycle A вҶ’ C вҶ’ V вҶ’ A; persist to `localStorage` key `trellis_variant_dev` | Pill text swap (no animation) |
| PlannerScreen hidden (Variant V) | `IntersectionObserver` ratio === 0 OR `document.visibilitychange` === hidden | Pause video element | None |
| PlannerScreen visible again (Variant V) | observer ratio > 0 AND document visible | Resume video playback | None |

**Single-tooltip-at-a-time rule (D-36):** Only one tooltip is ever open. Opening a new one closes any existing one.

**Touch accessibility:** All interactive leaves have a 44Г—44px hit target via an invisible padded `<rect>` SVG element (visual leaf can be 24вҖ“32px). Empty-state CTA button uses `Button size="md"` (44px+ height by default).

## States Specification

Each state must be implemented:

| State | When | Visual |
|-------|------|--------|
| **Loading** | Component first mounts, before data computed | Background asset visible (Variant A/C/V renders immediately); SVG overlay empty; vines start drawing as soon as data resolves |
| **Empty** | `anchors.length === 0` | Empty-state overlay: trellis background + small bud illustration on ground (`рҹҢұ` SVG, ~32px) + heading + body + CTA button. Centered over hero canvas with semi-transparent backdrop overlay (`rgba(255, 251, 245, 0.7)` light / `rgba(17, 17, 17, 0.7)` dark) so empty trellis is still partially visible |
| **Populated** | `anchors.length > 0` | Vines + leaves + blossoms + fruits per state computation. No overlay text. |
| **Error** | Data computation throws OR asset fails to load | Show error state heading + body. NO retry button вҖ” the existing PlannerScreen pull-to-refresh handles this |
| **Animating** | During state transition or mount | Animations described in Interaction Contract apply; leaves remain tappable throughout (interaction not blocked by animation) |
| **All-green celebration** | All anchors are green or better | Subtle bloom glow + ambient sway active in addition to populated state |
| **Tooltip open** | After leaf tap | Tooltip Card overlays canvas; canvas itself dims to `opacity: 0.85` to focus attention; leaf-of-interest stays at `opacity: 1` |
| **Variant V backgrounded** | PlannerScreen not visible | Video element `pause()` called; static last-frame poster shown via `poster` attribute |
