<objective>
Eliminate the root cause of persistent mindmap duplicates: asymmetric label matching, case-sensitive candidate extraction, and weak LLM reuse guidance in the incremental pipeline.

Purpose: After Phase 23 shipped the 3-step pipeline, duplicates still accumulate because:
- commitClassificationResult lowercases anchor titles (line 616) but compares cluster labels exactly (line 617) → mismatched capitalization creates duplicate clusters.
- extractUniqueBranches / extractClustersUnderBranch use `Set<string>` with exact strings → "Psychology" and "psychology" survive as parallel branches forever.
- buildStepPrompt lacks the reuse guidance that single-call classifyAndAnchor has ("Reuse existing... create new only when truly needed").
- Step 3 shows anchor names only, so the LLM can't distinguish "RL" from "Reinforcement Learning".

This plan fixes all of the above without a post-hoc merge pass. Plan 23-04 (in parallel) makes the user-triggered reorganize robust so historical duplicates can be cleaned up.

Output: symmetric label normalization throughout the pipeline, reuse-biased prompts, alias harvesting, new unit tests.

    ```typescript
    export function buildStepPrompt(
      level: 'branch' | 'cluster' | 'anchor',
      candidates: Array<string | { id: string; name: string; summary?: string }>,
    ): string {
      if (candidates.length === 0) {
        return `Select or create a ${level}.\n\nNo existing ${level}s yet — create a new one.\n\nRespond with {"index":"NEW","name":"<${level} name>"}.`;
      }

      const numbered = candidates
        .map((c, i) => {
          if (typeof c === 'string') return `${i}. ${c}`;
          const tail = c.summary ? ` — ${c.summary.slice(0, 80)}` : '';
          return `${i}. ${c.name}${tail}`;
        })
        .join('\n');

      return [
        `Select the best ${level} for this question.`,
        `Prefer reusing an existing ${level}. Only create NEW if no option reasonably fits.`,
        `Minor wording or case differences should reuse the existing ${level}.`,
        ``,
        `Existing ${level}s:`,
        numbered,
        ``,
        `Respond with the index number (0-${candidates.length - 1}) to select an existing ${level}, or {"index":"NEW","name":"<${level} name>"} to create a new one.`,
      ].join('\n');
    }
    ```

    **2. Add a coercion helper** (place near buildStepPrompt):
    ```typescript
    /**
     * If decision says NEW but the proposed name collides (by labelKey) with a candidate,
     * return the candidate index and the rejected name for alias harvesting.
     * Returns null if no coercion is needed.
     */
    export function coerceNewToExisting(
      decision: StepDecision,
      candidates: Array<string | { id: string; name: string }>,
    ): { selectedIndex: number; rejectedName: string } | null {
      if (!decision.isNew || !decision.newName) return null;
      const k = labelKey(decision.newName);
      if (!k) return null;
      for (let i = 0; i < candidates.length; i++) {
        const candName = typeof candidates[i] === 'string' ? candidates[i] as string : (candidates[i] as { name: string }).name;
        if (labelKey(candName) === k) {
          return { selectedIndex: i, rejectedName: decision.newName };
        }
      }
      return null;
    }
    ```
