<objective>
Stop the bottom-navigation bar flickering up/down when the keyboard opens on the Ask screen.
Root cause: `useKeyboard.handleResize` recomputes open/closed on every visualViewport.resize and
the single `MIN_KEYBOARD_HEIGHT=150` threshold crosses both directions during the Android WebView
keyboard-open animation, toggling `keyboardOpen` and reversing the nav's `y` spring.

Purpose: Keyboard open raises the input island smoothly with a single nav slide-down, no oscillation.
Output: Hysteresis/debounce in useKeyboard (lowest-risk Option A from RESEARCH); source-guard test.
</objective>
