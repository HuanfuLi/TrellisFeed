# Phase 55.1 Plan 09: GAP-B Render-Layer Trellis Animation Gate Summary

Route-gated the trellis leaf framer-motion animations so the always-mounted PlannerScreen does zero per-frame motion work while off-screen, removing the compositor starvation behind Home↔Planner↔Ask swipe + Home feed scroll lag (worst with Trellis Dev Mode ON, ~31 animated nodes).

## What Was Built

- **`trellis-animation-gate.ts`** — a React-free, framer-motion-free pure helper `shouldAnimateTrellis({ isPlannerActive, devMode?, nodeCount?, inView? })`. Contract: `isPlannerActive === false` → `false` (always, regardless of any composition input); `isPlannerActive === true` → `true`. The route check is the dominant lever for the cross-screen lag; the other inputs are accepted only so the canvas can compose with the existing `leafAnimationMask`. Mirrors `trellis-perf-mask.ts` so it unit-tests under `node --test` without a JSX transform.
- **`trellis-animation-gate.test.mjs`** — 6 executing assertions including the load-bearing GAP-B case `shouldAnimateTrellis({ isPlannerActive: false, devMode: true, nodeCount: 31, inView: true }) === false`, route-dominance, idempotency, and a source-read guard that the module imports no react/framer-motion.
- **TrellisHero** computes `animationsEnabled = shouldAnimateTrellis({ isPlannerActive, nodeCount })` (re-reads every render so it tracks navigation on the always-mounted slot) and passes it to `TrellisCanvas` as both `ambientEnabled` (preserving prior route-only sway behavior) and the new `animationsEnabled` master gate.
- **TrellisCanvas** removed the hardcoded `const inView = true`; `inView` now equals `animationsEnabled`, and `perfGuardActive` is forced true when `!animationsEnabled` (off-screen Planner treated as fully out-of-view for the perf mask). Threads `animationsEnabled` to each `TrellisLeaf`.
- **TrellisLeaf** accepts `animationsEnabled` (default true). When false: outer `motion.g` uses scalar `rotate={rotation}` (no sway keyframe array, no `repeat: Infinity`), `initial={false}` (no entry-spring re-trigger), and `transition={{ duration: 0 }}`; the inner pulse `motion.g` does not animate. Leaf STATE/shape/color is byte-identical in both cases — only WHETHER it animates changes.
