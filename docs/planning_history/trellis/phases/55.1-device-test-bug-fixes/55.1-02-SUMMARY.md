# Phase 55.1 Plan 02: Text-art Truncation On Provider/Locale Switch Summary

**Stop Gemini thinking-token starvation and shorter-regen overwrites from collapsing already-generated text-art posts to fragments ("T" / "Is your"), via a reject-fragment gate, an anti-truncation persist-merge guard, and a raised Gemini token budget + opt-in thinkingConfig disable.**

## Accomplishments
- `tightenTextArtContent` now rejects unusable single-token fragments (no internal whitespace AND length < 8 → `null`), so a starved Gemini regen falls back to `teaser.hook || title` instead of persisting "T".
- New pure `mergeTextArtContent(existing, incoming)` guard: never overwrites a non-empty `textArtContent` with an empty/shorter regeneration. `_persistStylesToCache` now routes through it (dropping the bare `info.textArtContent ?? p.textArtContent` that allowed trade-downs).
- Text-art Gemini call budget raised from `maxTokens: 80` to `512` and tagged `disableThinking: true`; `toGeminiPayload` emits `generationConfig.thinkingConfig.thinkingBudget: 0` only when opted in, so the model no longer spends its budget on thinking tokens.
- Three test files lock the contract: extended tightener reject assertions, new persist-merge contract + wiring guard, new Gemini budget/thinkingConfig source guard.

## Decisions Made
- **Field-shape verification (A2):** Context7 MCP / ctx7 CLI were unavailable in this worktree, so a live doc fetch could not be performed. The `generationConfig.thinkingConfig.thinkingBudget` field is the documented, stable Gemini v1beta shape (matches the 55.1-RESEARCH.md `discuss.ai.google.dev` citation). To honor the "don't ship an unverified field that errors the API" directive, the field is emitted **opt-in only** via the `disableThinking` flag on the text-art path — so even on a model that does not support it, the broad chat/classification/stream paths never send it. This is the safest interpretation of "apply BOTH defenses" while bounding blast radius. **Recommend the device-UAT gate confirm the live Gemini thinking model accepts the field (no API error) before considering A2 fully closed.**
- **A1 value:** 512 (conservative, matches research). The rendered card does not lengthen because `tightenTextArtContent` still caps to one sentence ≤ 80 chars.
- **Open Question 2 (keep+gate):** kept the LOCALE_CHANGED strip; the corruption-prevention is the gate + merge guard + budget, documented inline at the strip.
- **Fragment-reject scope:** the tightener rejects only single-token fragments (no internal space AND length < 8). The 2-word "Is your"-style truncation has an internal space and so passes the tightener, but is caught at persist time by `mergeTextArtContent`'s shorter-regen rule — the layered defense the plan intends.

## Small-maxTokens Audit (Open Question 3)
Audited all `maxTokens: <number>` call sites in `app/src/`:
- `concept-feed.service.ts` text-art — was 80, **fixed → 512 + disableThinking**.
- `canonical-knowledge.service.ts:1035` classification step (`maxTokens: 100`) and `:1137` (`maxTokens: 120`) — **NOT fixed.** These produce short structured selections (index/name), use JSON-mode/structured output that fails loudly (retry path at `:1725`) rather than silently persisting visible garbage, and have no operator-reported truncation. Per the plan ("fix only if it feeds a Gemini thinking path AND is truncating"), left as-is; flagged here for visibility. If a future device-UAT shows classification truncation on a Gemini thinking model, the same `disableThinking: true` flag can be added.
- Remaining sites (`8192`, `16384`) are well above the thinking-overhead floor.
