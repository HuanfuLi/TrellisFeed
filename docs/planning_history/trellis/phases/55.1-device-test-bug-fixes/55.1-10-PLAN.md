<objective>
GAP-E (BUGFIX-08, NEW FEATURE — spawned from GAP-C round 3). The post-body lazy-load makes the user
wait seconds for the model to THINK before the body streams into PostDetailScreen. The operator wants
a separate, user-configurable model entry used ONLY for latency-sensitive on-open generation, with
thinking/reasoning DISABLED for low streaming latency — distinct from the main Ask Q&A provider
(where reasoning is fine).

Purpose: add an optional "low-latency generation model" config; when set, route the on-open one-shot
generators (post body, news essay, post-context Q&A) through it with thinking off; when unset, fall
back to the current main model (zero behavior change). Disable thinking per provider only on this
fast-model path (must NOT regress the intentional Gemini text-art thinking-budget behavior, which is
a SEPARATE path).

Output: settings type + service + UI (Settings → AI), 4-bundle i18n strings, provider thinking-disable
plumbing, on-open generators routed through a `resolveGenerationConfig` helper, executing tests, and a
mandatory on-device UAT (configure a non-reasoning model, tap into a post, confirm immediate stream).

Decisions to confirm at the first checkpoint:
- Config shape: a separate full provider entry (provider + model + key) vs a model-override on the
  existing provider. RECOMMENDATION: separate optional `fastModel` config object (mirrors LLMConfig
  shape, defaults to unset/disabled), so users can point it at a wholly different provider/model/key.
- In-scope call sites. RECOMMENDATION: post body (generateStandardEssay/generateVideoEssay/
  generateTextArtEssay), news essay (generateNewsEssay), and post-context Q&A. OUT of scope: Ask Q&A,
  classification, planner, podcast, flashcard.
</objective>
