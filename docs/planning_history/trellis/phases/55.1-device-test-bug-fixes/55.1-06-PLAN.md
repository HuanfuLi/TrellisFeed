REVIEW_COMPLETED, GRAPH_UPDATED, ANCHOR_DELETED, HARVEST_COMPLETED and runs a FULL recompute on each.
From app/src/services/trellis-state.service.ts:
```
export function computeLeafState(anchor, qaChildren, blossomSinceDate, fcMap): LeafState
export function buildTrellisState(questions: Question[]): TrellisLayout
```
buildTrellisState builds the anchor reflection tree, builds an fcMap from flashcardService.getAll(),
then iterates EVERY anchor + legacy node calling computeLeafState per node (lines 227-284) and computing
leaf positions. It also persists blossom dates as a side effect during the loop (setBlossomDate /
clearBlossomDate) — a write inside the hot loop.
From app/src/components/trellis/TrellisCanvas.tsx: renders `nodes.map(...)` into `<TrellisLeaf>` three
times (vine attach paths line 109, a filtered subset line 133, all leaves line 152). There is already a
`perfGuardThresholdExceeded` + `leafAnimationMask` mechanism (trellis-perf-mask) gating one-shot
animations on large canvases. TrellisLeaf is NOT memoized.
From app/src/screens/PlannerScreen.tsx: `const { layout } = useTrellisData()`; derives deadNodes/dyingNodes
via `layout.nodes.filter(...)` (recomputed every render), passes `nodes={layout.nodes}` to the canvas.
