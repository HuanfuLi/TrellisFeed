<objective>
Root-cause and fix the intermittent curiosity-feed buffer-queue under-refill (TUNE-03): swipe-for-more sometimes yields 1, 4, or 0 new posts instead of the intended 8 even though the derived list still has unread capacity. The bug lives in the size-check + refill-threshold + walker-batch + mutex-timing interaction across `concept-feed.service.ts:generateMorePosts`/`refillQueue` and `post-queue.service.ts` (`needsRefill`, `dequeue`, `enqueueInterleaved`, `walkDerivedList`).

Purpose: TUNE-03 requires the refill be reliable AND root-caused — the operator's standing rule forbids a hypothesis-only fix on a device/runtime bug. This plan reproduces the under-refill with dev-gated browser instrumentation FIRST (D-01/D-02 evidence bar), then fixes the confirmed cause, then freezes the corrected 8-post batch behavior as a regression test.
Output: dev-gated refill-cycle instrumentation; the confirmed-cause fix in the two services; `refill-reliability.test.mjs` reproducing the under-refill and asserting the corrected 8-post batch; existing `derived-list.test.mjs` + `refill-queue-integration.test.mjs` still green.
</objective>
