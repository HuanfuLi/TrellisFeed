<objective>
Wire the locale into the root DOM and onboarding. Adds locale-specific font stacks via CSS `--font-sans` + `:root[data-locale]` (D-23), inserts a language-confirmation step into `OnboardingScreen` before the consent step (D-18), and registers the `LOCALE_CHANGED` AppEvent so Plan 04 can subscribe to it for mid-stream abort (D-22 wiring).

Purpose: Per D-18 — user MUST confirm (or override) detected locale before the rest of onboarding runs in that locale. Per D-23 — CJK scripts need locale-appropriate system fonts without webfont downloads. Per D-22 — a LOCALE_CHANGED event enables the mid-stream cancellation in Plan 04 and any future subscribers that need imperative re-fetch.

Output: First-launch users land on a language step labelled in all 4 scripts; font stack adapts automatically; event-bus gains the locale-change signal.
</objective>

    ```typescript
    import i18n from '../locales';
    import { detectDeviceLocale } from '../lib/locale';
    import type { SupportedLocale } from '../types';
    import { eventBus } from '../lib/event-bus';

    // inside OnboardingScreen():
    const [selectedLocale, setSelectedLocale] = useState<SupportedLocale>(() => {
      // synchronous best-effort; async refinement below
      const cur = i18n.language;
      return (['en','zh','es','ja'] as const).includes(cur as SupportedLocale)
        ? (cur as SupportedLocale)
        : 'en';
    });

    useEffect(() => {
      detectDeviceLocale().then((detected) => setSelectedLocale(detected));
    }, []);
    ```
