    **File 1: `app/src/lib/swipe-tab-context.ts`**

    Create a React context that shares two things:
    - `swipeProgress`: a `MotionValue<number>` representing fractional screen index (0=Home, 1=Planner, 2=Ask, 3=Graph, 4=Settings). Used by BottomNavigation for real-time color interpolation without re-renders.
    - `navigateToTab`: a function `(targetIndex: number) => void` that triggers slide animation to a specific tab. Used by BottomNavigation for tab-tap animation (D-04).

    ```typescript
    import { createContext, useContext } from 'react';
    import type { MotionValue } from 'framer-motion';

    interface SwipeTabContextValue {
      swipeProgress: MotionValue<number>;
      navigateToTab: (targetIndex: number) => void;
    }

    export const SwipeTabContext = createContext<SwipeTabContextValue | null>(null);

    export function useSwipeTab(): SwipeTabContextValue {
      const ctx = useContext(SwipeTabContext);
      if (!ctx) throw new Error('useSwipeTab must be used within SwipeTabContainer');
      return ctx;
    }
    ```

    **File 2: `app/src/components/SwipeTabContainer.tsx`**

    Create the swipe container component. Props:
    ```typescript
    interface SwipeTabContainerProps {
      screens: React.ReactNode[];       // Array of 5 screen elements
      routes: readonly string[];        // ['/home', '/planner', '/ask', '/graph', '/settings']
      children?: React.ReactNode;       // Rendered outside the strip (BottomNavigation, overlays)
    }
    ```

    Internal state/refs (NOT useState for performance — use refs):
    - `activeIndexRef = useRef(0)` — current committed screen index
    - `screenWidthRef = useRef(window.innerWidth)` — read fresh in onPanStart
    - `lockAxisRef = useRef<'x' | 'y' | null>(null)` — axis lock state per gesture
    - `gestureBlockedRef = useRef(false)` — set in onPanStart if data-no-swipe-nav detected
    - `dragOffset = useMotionValue(0)` — pixel offset during drag
    - `keyboardOpen` state via focusin/focusout listener (D-09)

    Derived motion values:
    - `stripX = useTransform(dragOffset, (raw) => -(activeIndexRef.current * screenWidthRef.current) + raw)`
    - `swipeProgress = useTransform(stripX, (x) => -x / screenWidthRef.current)`

    Route sync:
    - `useLocation()` to detect external navigation (tab tap, back button, programmatic navigate)
    - `useEffect` watching `location.pathname`: find matching index in routes array, if different from activeIndexRef.current, update activeIndexRef.current and animate dragOffset to 0 (this handles initial load and Android back button)
    - Only call `navigate()` inside `onPanEnd` after threshold commit — never during drag (per Pitfall 2)

    `onPanStart` handler:
    1. Reset lockAxisRef to null
    2. Read `screenWidthRef.current = window.innerWidth` (per Pitfall 6)
    3. Check `(e.target as HTMLElement).closest('[data-no-swipe-nav]')` — if found, set gestureBlockedRef.current = true and return (per D-08, D-10)
    4. Otherwise set gestureBlockedRef.current = false

    `onPan` handler:
    1. If `shouldBlockGesture({keyboardOpen, gestureBlocked: gestureBlockedRef.current})` return
    2. If lockAxisRef.current is null, call `resolveAxisLock({x: info.offset.x, y: info.offset.y})` — set lockAxisRef if result is non-null
    3. If lockAxisRef.current !== 'x', return (vertical scroll wins)
    4. Call `computeDragOffset(info.offset.x, activeIndexRef.current, routes.length)` and set result to `dragOffset`

    `onPanEnd` handler:
    1. If gestureBlockedRef or lockAxisRef !== 'x', animate dragOffset to 0 and return
    2. Call `resolveCommitIndex(info.offset.x, activeIndexRef.current, screenWidthRef.current, routes.length)`
    3. If newIndex !== activeIndexRef.current:
       - Set `activeIndexRef.current = newIndex`
       - Call `navigate(routes[newIndex])` (URL sync on commit only)
    4. Animate `dragOffset` to 0 with spring: `animate(dragOffset, 0, { type: 'spring', stiffness: 300, damping: 30, mass: 0.8 })` (per D-16: ~250ms spring)

    `navigateToTab(targetIndex)` function (for D-04, D-05):
    1. If targetIndex === activeIndexRef.current, return
    2. Calculate visual jump: `const jump = (activeIndexRef.current - targetIndex) * screenWidthRef.current`
    3. Set `activeIndexRef.current = targetIndex`
    4. Set `dragOffset.set(jump)` (instant visual position at old screen)
    5. Animate `dragOffset` to 0 with spring (same params as onPanEnd)
    6. Call `navigate(routes[targetIndex])`
    7. Per D-05: slide directly from current to target — no intermediate screens

    Keyboard detection (D-09):
    - `useState(false)` for keyboardOpen
    - `useEffect` with `focusin`/`focusout` listeners on document
    - `focusin`: check if target is INPUT, TEXTAREA, or isContentEditable — set true
    - `focusout`: set false

    Render:
    ```tsx
    <SwipeTabContext.Provider value={{ swipeProgress, navigateToTab }}>
      <motion.div
        onPanStart={onPanStart}
        onPan={onPan}
        onPanEnd={onPanEnd}
        style={{
          display: 'flex',
          width: `${routes.length * 100}vw`,
          x: stripX,
          touchAction: 'pan-y',  // allow vertical scroll, capture horizontal via onPan
        }}
      >
        {screens.map((screen, i) => (
          <div
            key={routes[i]}
            style={{
              width: '100vw',
              flexShrink: 0,
              minHeight: '100vh',
              overflow: 'hidden',
            }}
          >
            {screen}
          </div>
        ))}
      </motion.div>
      {children}
    </SwipeTabContext.Provider>
    ```

    Export: `export function SwipeTabContainer({ screens, routes, children }: SwipeTabContainerProps)`
