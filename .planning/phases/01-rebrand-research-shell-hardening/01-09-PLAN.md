<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| participant → hidden researcher page | A participant holding the device must not reach account/condition edits or destructive controls; the PIN is a UI-concealment gate only. |
| local records → recovery export | Export is per-participant recovery, not cross-account aggregation. |

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |

| T-01-29 | Elevation of privilege | diagnostics access | medium | mitigate | SHA-256 PIN gate; documented as a local UI gate only; no server power granted (RESEARCH A3/Q4). |
| T-01-30 | Tampering | destructive/account controls on device | high | mitigate | Diagnostics exposes status + recovery export only; no clear/reset/account/condition/API-key control (surface test asserts). |
| T-01-31 | Information disclosure | recovery export scope | medium | mitigate | Export reads only local `research_records` (per-participant); no central aggregation on device (D-11). |
| T-01-32 | Information disclosure | secret in Settings/onboarding | high | mitigate | No key control in Settings; onboarding credentialing resolved by decision checkpoint; no real key committed (D-04/D-12). |
</threat_model>
