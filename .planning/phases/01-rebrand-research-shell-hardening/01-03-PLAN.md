Stand up the collection backend: a dependency-light Cloudflare Worker + D1 database with two public routes — install-resolve (numeric account → fixed condition/topic) and bounded batch ingest with idempotent, parameterized writes. This is the server half of LOG-01 and locked decisions D-13 (real deployed collector, at-least-once upload with server ACK) and D-02/D-08 (server owns condition/topic; strict field allowlist). Package installs require a human legitimacy checkpoint before proceeding (wrangler flagged [SUS] in the RESEARCH Package Legitimacy Audit).

Purpose: Only the server can declare a record accepted; idempotent writes make client retries safe under lost responses.
Output: A locally testable Worker (fake-D1 unit tests) with schema, validation, and idempotent ingest; deploy + admin/export land in Plan 05.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| participant app → public ingest endpoint | Untrusted JSON crosses here; may carry injection, prohibited fields, oversized bodies, or spoofed condition/topic. |
| Worker → D1 | Parameterized writes only; participant text must never be interpolated into SQL. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-01-07 | Tampering | ingest SQL construction | high | mitigate | `prepare().bind()` / `batch()` only; no string interpolation of question/account text (validation.test asserts). |
| T-01-08 | Tampering | client-supplied condition/topic | high | mitigate | Re-derive condition/topic from `study_accounts`; reject unknown accounts (ingest.test asserts). |
| T-01-09 | Repudiation | lost-response duplicate delivery | medium | mitigate | Immutable event id + INSERT OR IGNORE + Q/A revision compare; ACK = durably stored (ingest.test asserts). |
| T-01-10 | Information disclosure | prohibited fields (URL/device/payload) | high | mitigate | Server field allowlist rejects any non-allowlisted key (validation.test asserts a buried payload fails). |
| T-01-11 | Denial of service | oversized/malformed public ingest | high | mitigate | Bound body ≤256 KiB / ≤100 records; strict content-type; reject before DB work. |
| T-01-SC | Tampering | wrangler / fflate npm installs | high | mitigate | Blocking human legitimacy checkpoint before install; pin audited versions; no `[SLOP]` packages. |
