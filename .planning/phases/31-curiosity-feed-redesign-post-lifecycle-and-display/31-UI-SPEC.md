## Spacing Scale

Declared values (existing project tokens from `index.css`, Phase 28 D-26):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`--space-xs`) | Icon gaps, inline padding |
| sm | 8px (`--space-sm`) | Compact element spacing, checklist item gaps |
| md | 12px (`--space-md`) | Inner card padding |
| lg | 16px (`--space-lg`) | Default element spacing, vine card padding |
| xl | 20px (`--space-xl`) | Section padding |
| 2xl | 24px (`--space-2xl` / `--section-gap`) | Section breaks, history day-group gaps |
| 3xl | 32px (`--space-3xl`) | Page-level spacing |

**Note on non-standard values:** 12px (`--space-md`) and 20px (`--space-xl`) are inherited project tokens established in Phase 28 D-26 (`index.css`). These predate Phase 31 and are reused unchanged. Both are multiples of 4 and do not break grid alignment. Phase 31 introduces no new spacing values outside the standard set (4, 8, 16, 24, 32, 48, 64).

Exceptions:
- Scroll-to-top FAB: 44x44px minimum touch target (WCAG 2.5.5)
- Bottom offset for scroll-to-top FAB: 96px from bottom edge (clears BottomNavigation + safe area)

## Component Inventory

### 1. VineProgress (replaces ConceptProgressCard + CompactProgressBar)

**Two display modes, same component:**

| Mode | Trigger | Container |
|------|---------|-----------|
| Inline card | Visible between bento grid and feed | `var(--card)` background, `var(--radius)` corners, `var(--shadow-1)`, 16px padding |
| Compact header | Inline card scrolled out of viewport | Sticky header bar, `var(--card)` bg, `var(--shadow-1)`, 12px vertical padding, 16px horizontal |

**Vine illustration spec:**
- Orientation: horizontal, left-to-right
- Left anchor: potted plant icon (24x24px), always visible
- Vine stem: 3px stroke, color `var(--primary-40)`, grows rightward proportional to explored/total ratio
- Leaves: small SVG shapes (8x12px) sprouting at even intervals along vine, fill `var(--primary-40)` at 60% opacity
- Flowers: 10x10px circles at concept milestones, fill `var(--primary-40)` when explored, `var(--muted)` when unexplored
- Completion state: all flowers bloom (scale 1.0 to 1.3 pulse, 600ms ease-in-out), fruit icons (12x12px) appear, vine color shifts to `#E8A838`
- Total vine height: 48px in inline mode, 36px in compact mode

**Expand/collapse interaction:**
- Tap vine area to expand concept checklist below
- Down-arrow indicator: `ChevronDown` from lucide-react, 16px, `var(--muted-foreground)`, rotates 180deg on expand
- Collapse: tap vine again or tap outside (click-away listener)
- Animation: height auto with 200ms ease-out transition

**Concept checklist (expanded state):**
- Vertical list below vine, 8px top margin
- Each item: concept name as text, 14px weight 400, `var(--foreground)`
- Explored items: strikethrough, `var(--muted-foreground)` color
- Unexplored items: tappable, navigates to first post for that concept in feed
- Item height: 36px (touch-friendly row)
- Max visible items before scroll: 6 (container max-height 216px, overflow-y auto)

**Hidden state (D-07):** When no concepts are due, render `null`. No empty placeholder.

**Update timing (D-06):** Re-renders on Home screen mount/focus, not real-time.

**History icon:** `Clock` from lucide-react, 20px, `var(--muted-foreground)`, positioned top-right of vine inline card. `aria-label="View post history"` (i18n: `home.history.iconLabel`). Tappable, navigates to `/history`.

### 2. SuggestionCard (new post type)

**Card layout:**
- Background: `var(--card)` with `var(--shadow-1)`
- Border radius: `var(--radius)`
- Padding: 16px
- Total height: approximately 280px (matches image post card height)

**Header:**
- Label: "You may also like" (i18n key: `home.feed.suggestionTitle`)
- Font: 14px weight 600, `var(--foreground)`
- Sparkles icon (lucide-react `Sparkles`, 16px) left of label, color `var(--primary-40)`

**Topic buttons (3 items):**
- Vertical stack, 8px gap between items
- Each button: full-width, 48px min-height, `var(--surface-variant)` background, `var(--radius)` corners
- Text: 14px weight 400, `var(--foreground)`, left-aligned with 16px horizontal padding
- Right chevron: `ChevronRight` 16px, `var(--muted-foreground)`
- Hover/press: background shifts to `color-mix(in srgb, var(--primary-40) 8%, var(--surface-variant))`
- Tap action: navigate to Ask screen, send topic as question

**Non-interactive card body:** Tapping card background is a no-op. Only topic buttons respond to tap.

### 3. InlineVideoPlayer (landscape videos in feed)

**Container:**
- 16:9 aspect ratio, `border-radius: var(--radius)`, overflow hidden
- Poster: YouTube thumbnail (existing `post.youtubeUrl` thumbnail derivation)
- Play button overlay: centered, 48x48px semi-transparent circle (`rgba(0,0,0,0.6)`), white triangle icon

**Playback behavior:**
- Tap play: embed YouTube iframe at same position, autoplay
- No essay/summary generation on play (D-28)
- Stop on swipe-away: listen for screen visibility changes, remove iframe on hidden

**Controls:** Native YouTube iframe controls (no custom controls)

### 4. ScrollToTopFAB

**Appearance:**
- Position: fixed, bottom-right corner
- Bottom offset: 96px (above BottomNavigation)
- Right offset: 16px
- Size: 44x44px circle
- Background: `var(--card)` with `var(--shadow-2)`
- Icon: `ArrowUp` from lucide-react, 20px, `var(--primary-40)`
- Border: 1px solid `var(--border)`

**Visibility:**
- Hidden by default (opacity 0, pointer-events none)
- Appears after scrolling 400px from top (fade in, 200ms)
- Disappears within 100px of top (fade out, 200ms)

**Action:** `scrollTo({ top: 0, behavior: 'smooth' })`

### 5. PostHistoryScreen

**Entry points:**
- History icon near vine card area: `Clock` from lucide-react, 20px, `var(--muted-foreground)`, positioned top-right of vine inline card, `aria-label="View post history"` (i18n: `home.history.iconLabel`)
- Settings > Data section: tappable row "Post History"

**Route:** `/history` (sub-screen overlay, same pattern as `/review`, `/podcast`)

**Layout:**
- Header: standard `Header` component with `backTo` prop, title "Post History" (i18n: `home.history.title`)
- Content: vertical list grouped by day

**Day group:**
- Date heading: 14px weight 600, `var(--foreground)`, sticky top, `var(--background)` bg
- Format: "Today", "Yesterday", or locale-formatted date (e.g., "Apr 15")
- 24px bottom margin after each group

**Post item in history:**
- Card style: `var(--card)` bg, `var(--radius)` corners, `var(--shadow-1)`, 12px padding
- Left: thumbnail (48x48px, rounded 8px) — post image, video thumbnail, or text-art color swatch
- Right: post title (14px weight 400, `var(--foreground)`, 2-line clamp), concept label below (12px, `var(--muted-foreground)`)
- Tap: navigate to PostDetailScreen
- Item gap: 8px between items

**Error state:**
- Displayed when history data fails to load from localStorage
- Layout: centered in content area, 200px height
- Icon: `AlertCircle` from lucide-react, 32px, `var(--muted-foreground)`
- Message: "Couldn't load history" (i18n: `home.history.errorTitle`), 14px weight 400, `var(--muted-foreground)`, centered
- Action: "Try again" button (i18n: `home.history.errorRetry`), 14px weight 600, `var(--primary-40)`, tappable, retriggers data load

### 6. BotanicalLoadingState (empty queue)

**Displayed when:** Queue is empty and generation is in progress.

**Layout:**
- Centered in feed area, 200px height
- Illustration: simple botanical line art (potted seedling, 64x64px), drawn with `var(--primary-40)` stroke, 2px
- Below illustration: "Growing your feed..." (i18n: `home.feed.loadingTitle`), 14px weight 400, `var(--muted-foreground)`, centered
- Subtle pulse animation on illustration (opacity 0.5 to 1.0, 1.5s ease-in-out infinite)

**Generation fallback error state:**
- Displayed when generation itself fails (all fallbacks exhausted, or network/LLM error)
- Layout: replaces the botanical loading illustration in the same 200px centered area
- Icon: `AlertCircle` from lucide-react, 32px, `var(--muted-foreground)`
- Message: "Couldn't generate posts" (i18n: `home.feed.generationErrorTitle`), 14px weight 400, `var(--muted-foreground)`, centered
- Sub-message: "Check your API keys in Settings" (i18n: `home.feed.generationErrorBody`), 12px weight 400, `var(--muted-foreground)`, centered, 4px top margin
- Action: "Retry" button (i18n: `home.feed.generationErrorRetry`), 14px weight 600, `var(--primary-40)`, tappable, retriggers queue refill

**Feedback button:**
- Below loading text (or below error action), 16px top margin
- Text: "Posts not interesting?" (i18n: `home.feed.feedbackPrompt`), 12px weight 400, `var(--primary-40)`, underline
- Tap: opens device email client via `mailto:` link with pre-filled to and subject

## Interaction Contracts

### Pull-up-for-more
- Reuse existing `PullUpHint` component unchanged
- Serves 4 posts per pull (changed from 6)
- Queue auto-refills at < 8 posts (background, non-blocking)

### Video stop on swipe-away (D-29)
- Listen for `SwipeTabContainer` active index changes
- When Home screen loses focus: pause/remove any active video iframe
- Implementation: ref-based cleanup on visibility change

### Vine tap-to-expand
- Single tap toggles checklist visibility
- Outside-tap collapses (event listener on document, cleanup on unmount)
- No drag gestures on vine

### Concept tap in checklist (D-04)
- Scroll feed to first post matching that concept
- Use `scrollIntoView({ behavior: 'smooth', block: 'center' })`
- If concept post not yet in rendered feed, no-op (concept is queued for future batch)

### Warm start (D-30)
- On new day: render yesterday's remaining queue immediately
- Background: start generating today's posts
- Pull-up-for-more serves from today's queue once ready

### Scroll-to-top threshold
- Appear: 400px scroll distance from top
- Disappear: scroll position < 100px
- Transition: opacity 0 to 1, 200ms
