<task type="auto">
  <name>Task 2: Add touch overlay and close button on playing video iframes</name>
  <files>app/src/components/InfoFlow.tsx</files>
  <action>
MAJOR 6 fix: YouTube iframes capture ALL touch events, preventing scroll and trapping the user. Add a transparent overlay with a close/stop button on top of playing iframes (both landscape and short).

For BOTH the landscape video iframe (lines 286-293) and the short video iframe (lines 359-369), wrap the iframe in a container and add an overlay:

```tsx
<div style={{ position: 'relative', width: '100%', aspectRatio: '16 / 9' /* or '9/16' for shorts */, overflow: 'hidden' }}>
  <iframe
    src={`https://www.youtube.com/embed/${post.videoMeta.videoId}?autoplay=1&playsinline=1&rel=0`}
    style={{ position: 'absolute', inset: 0, width: '100%', height: '100%', border: 'none', pointerEvents: 'auto' }}
    allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture"
    allowFullScreen
    title={normalizedTitle || t('infoFlow.postImageAlt')}
  />
  {/* Transparent overlay — prevents iframe from capturing scroll touches */}
  <div
    style={{
      position: 'absolute',
      inset: 0,
      zIndex: 2,
      pointerEvents: 'auto',
      background: 'transparent',
    }}
    onClick={(e) => {
      e.stopPropagation();
      // Tap overlay = stop video
      setVideoPlaying(null);
    }}
  />
  {/* Close button — visible affordance to stop */}
  <button
    onClick={(e) => {
      e.stopPropagation();
      setVideoPlaying(null);
    }}
    style={{
      position: 'absolute',
      top: 8,
      right: 8,
      zIndex: 3,
      width: 32,
      height: 32,
      borderRadius: '50%',
      background: 'rgba(0,0,0,0.6)',
      border: 'none',
      color: 'white',
      display: 'flex',
      alignItems: 'center',
      justifyContent: 'center',
      cursor: 'pointer',
      padding: 0,
    }}
    aria-label="Stop video"
  >
    <X size={18} />
  </button>
</div>
```

Note: The overlay sits OVER the iframe, so `pointerEvents: 'auto'` on the overlay intercepts touches. The user taps overlay = video stops. The iframe still plays audio/video underneath. This prevents scroll interference entirely because the iframe never receives touch events.

The tradeoff: user cannot interact with YouTube player controls (play/pause/seek within the iframe). This is acceptable because:
- Tap to play, tap overlay to stop is the intended UX
- The video auto-plays on start
- Full YouTube interaction is available via the video's detail page

Import `X` from lucide-react if not already imported (it IS already imported at line 2).
