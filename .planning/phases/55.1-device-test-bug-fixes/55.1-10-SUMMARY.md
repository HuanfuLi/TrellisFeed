# Phase 55.1 Plan 10: Optional Low-Latency Generation Model (GAP-E) Summary

Adds an optional, user-configurable low-latency generation model (`fastModel`) that the on-open one-shot generators stream from with thinking/reasoning disabled, so post bodies start streaming immediately on tap-in instead of stalling on the main model's reasoning. Falls back to the main model with byte-identical requests when unset — zero behavior change for users who never configure it.

## What Was Built

- **`FastModelConfig` type** (`types/index.ts`): extends `LLMConfig` with an `enabled: boolean` master gate; optional `AppSettings.fastModel` field.
- **Default + deepMerge defaulting** (`settings.service.ts`): a disabled `fastModel` default lives in `defaultSettings`. Because `deepMerge` iterates `Object.keys(defaults)`, the key MUST be present there for pre-feature stored settings to receive the default — this is the additive-optional defaulting path, not a migration.
- **`resolveGenerationConfig(settings)`** (`generation-config.ts`, React-free): returns `{ config: fastModel, disableThinking: true }` when the fast model is `enabled` AND configured (key present, or baseUrl for local/lmstudio); otherwise `{ config: settings.llm, disableThinking: false }`. Strips the `enabled` flag so a clean `LLMConfig` is passed to `chatStream`.
- **Per-provider thinking-disable** (`providers/llm/index.ts`):
  - OpenAI `openAIStream`: adds `reasoning_effort: 'minimal'` to the request body only when `disableThinking` AND `provider === 'openai'` (skips local/lmstudio whose OpenAI-compatible servers vary).
  - Gemini `geminiStream`: now forwards `options.disableThinking` (and maxTokens/jsonMode) into `toGeminiPayload`, reusing the existing `thinkingConfig.thinkingBudget: 0` plumbing — previously the stream path dropped all options.
  - Anthropic `claudeStream`: documented no-op — extended thinking is opt-in (a `thinking:{}` block) and we never send one, so Claude already streams without it.
- **On-open generators routed** (`post-essay.service.ts`, `post-context-qa.service.ts`): `generateStandardEssay`, `generateVideoEssay`, `generateNewsEssay`, `generateTextArtEssay`, and post-context Q&A resolve `{config, disableThinking}` and pass them to `chatStream`. `generateEssayMeta` intentionally stays on the main model (out of scope).
- **Settings UI** (`SettingsAIScreen.tsx`): a new "Fast Generation Model" section (Zap icon) with an enable toggle, provider select, model field, and key field (password reveal), wired to `settingsService.set('fastModel', ...)`. Fields render only when enabled.
- **i18n**: `settings.sections.fastModel`, `settings.fields.fastModelEnabled`, `settings.descriptions.{fastModelBlurb,fastModelEnabled,fastModelHint}`, `settings.toast.fastModelSaved` added to all 4 bundles (en/zh/es/ja). Identifiers (OpenAI/Claude/Gemini/reasoning_effort/model names) kept verbatim.
- **Tests** (`fast-model-routing.test.mjs`): 6 executing resolver assertions (unset/disabled/unconfigured fall back; enabled+configured routes with disableThinking and strips `enabled`; local needs only baseUrl) + provider thinking-disable source guards + type/default/wiring guards + a negative guard that concept-feed is untouched.

## Critical Invariants Preserved

- **News defer-to-streamer**: this plan only changes WHICH model streams the news body on open. `concept-feed.service.ts` is NOT modified; `bodyMarkdown: ''` defer pattern intact (verified: 2 occurrences, file out of git diff).
- **Text-art Gemini budget**: the separate text-art thinking-budget-raise path in concept-feed is untouched; `gemini-text-art-budget.test.mjs` still passes.
- **No normalize() for optional additive field**: `fastModel` defaulted via deepMerge, no migration framework.
- **Ask-chat system prompt byte-stability** (`useQuestions.ts`): not touched — Ask Q&A keeps the main model.
- **Fallback is byte-identical**: when fastModel unset, `resolveGenerationConfig` returns the exact `settings.llm` object reference with `disableThinking:false`.
