# Phase 55.1 Plan 01: Cross-Session LLM Response Binding Summary

Bind a streaming Ask answer to the session it was requested in: persist by origin session id, gate the UI/overlay on origin-equals-active, and abort the in-flight stream on every session switch — closing the cross-session leak where a slow stream from session A rendered/persisted under session B's question.

## What was built

**Task 1 (commit 775d7efd) — Wave 0 scaffold.** Extracted a pure `resolvePersistTarget({ originSessionId, activeSessionId })` helper into a new sibling module `app/src/screens/ask-persist-target.ts`, returning `{ persistSessionId: originSessionId, updateUI: originSessionId === activeSessionId }`, and re-exported it from `AskScreen.tsx`. Added `app/tests/screens/AskScreen.session-binding.test.mjs` with (a) two contract cases for the helper and (b) seven source-reading guards. At end of Task 1 the contract cases were green and the source-guards were red, as designed.

**Task 2 (commit 1f962b2a) — origin-gated persist + abort-on-switch.** In `generateAiReply`:
- Capture `const originSessionId = sessionRef.current.id` up-front, before any streaming.
- Persist block now re-reads `sessionService.getById(originSessionId)` (was bare `sessionRef.current` — the leak) and bails if the origin session was deleted mid-stream.
- React-state updates (`setSession`/`setStreaming`) and both streaming-overlay paint sites are gated on `sessionRef.current.id === originSessionId` (via `resolvePersistTarget(...).updateUI` for the final commit).
- Removed the `generatingRef` early-return so a new session's send is honored rather than dropped.
- The `finally`/`catch` blocks now run only when `abortRef.current === controller`, so an aborted prior run (A) cannot clear the flag/overlay/abortRef that a newer run (B) already owns.

Wired `abortRef.current?.abort()` into `handleNewChat`, `handleSelectSession`, and the delete-active branch of `handleDeleteSession`, each before `setSession(...)`.

## Threat model

- **T-55.1-01 (Information disclosure) — mitigated:** persist re-read by `originSessionId`, never `sessionRef.current`.
- **T-55.1-02 (Tampering) — mitigated:** abort-before-switch in all three handlers discards the half-streamed placeholder.
- **T-55.1-03 (filter integrity) — accept-unchanged honored:** fix is AskScreen-caller-only; the Phase 55 RAW-ARGMAX malicious gate and its order vs `chatStream` were not touched.
