<objective>
Bind a streaming LLM answer to the session it was requested in. Fix the cross-session
leak where `generateAiReply` persists to the currently-active session (`sessionRef.current`)
instead of the originating session, never aborts the in-flight stream on session switch, and
silently drops the new session's send via `generatingRef`.

Purpose: Stop a prior session's answer from rendering/persisting under a different session's
question (observed: a 3rd-session answer under a 5th-session question).
Output: Origin-gated persist + UI in AskScreen, abort-on-switch wiring, and a regression test.
</objective>
