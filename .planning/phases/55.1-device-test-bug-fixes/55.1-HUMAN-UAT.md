### 2. BUGFIX-02 — Text-art survives provider/locale switch (live Gemini)
expected: With a Gemini thinking model configured and text-art posts already generated, switch provider to Gemini and/or change locale (incl. zh/ja). Existing text-art posts keep full sentences — no collapse to a few words / single token, no API error from `thinkingConfig.thinkingBudget:0`. Short CJK headlines are NOT discarded.
result: passed (device UAT 2026-05-21, round 2 — confirmed).

### 3. BUGFIX-03 — No bottom-nav flicker on keyboard open
expected: On the Ask screen (Android), tap the input. The input island raises smoothly and the bottom-navigation bar does NOT oscillate up/down during the keyboard transition (single settle, no flicker).
result: nav-bar behavior PASSED after the focus-driven instant-hide re-fix (commit 6fb7b325). PARTIAL overall — re-test surfaced a new issue: the text input bar "teleports" to its keyboard-open position with no animation (native adjustResize reflow). Tracked as GAP-A for gap closure.

### 4. BUGFIX-04 — Send fires on first tap with keyboard open
expected: On the Ask screen with the keyboard open, tap Send once. The message sends on the FIRST tap (not consumed by keyboard dismissal), fires exactly once, and the keyboard stays open.
result: passed (device UAT 2026-05-21)

### 5. GAP-A (BUGFIX-05) — ChatInput animates smoothly to keyboard-open position (no teleport)
expected: On the Ask screen (Android), tap the text input. The input bar should SLIDE/settle into its keyboard-open position over a short ease, not instantly jump/teleport.
result: **FAILED (round 3, 2026-05-21).** Input bar still teleports with no animation; nav bar behavior is good. The eased-`translateY`-decay approach (55.1-05) does not survive Android's native `adjustResize` reflow timing — the offset is applied and removed within the same paint as the native reflow, so no transit is visible. REOPENED → needs a different approach (drive position continuously off `visualViewport` resize events, or `interactive-widget`/CSS-keyboard-inset) — likely a research/debug cycle.

### 6. GAP-B (BUGFIX-05) — Planner trellis stays smooth at scale
expected: Smooth swipe + feed scroll with a large trellis.
result: **FAILED — worse (round 3, 2026-05-21).** Severe perf: laggy horizontal swipe between Home↔Planner↔Ask, and Home feed vertical swipe gets cut off from render overhead. **Operator's key clue: triggered by enabling Trellis Dev Mode; with it OFF the issue does not occur.** Root-cause reframe: Trellis Dev Mode (`trellis_dev_mode`) makes `buildTrellisState` return `buildDevTrellisState()` at trellis-state.service.ts:175 — which BYPASSES the blossom-date batching that 55.1-06 optimized. The dev build is ~31 static nodes (cheap); the cost is RENDER, not build: many simultaneously-animated leaf states (blossom/fruit/falling) via framer-motion, on the always-mounted PlannerScreen, keep animating while off-screen and steal the compositor during other screens' swipes. `TrellisCanvas.tsx:156` hardcodes `inView = true` (IntersectionObserver never wired); only ambient sway is gated on `isPlannerActive`. 55.1-06 measured the wrong code path. REOPENED → fix the render/animation layer (pause trellis animations when Planner is off-screen; wire real visibility into `inView`). Applies to real users with many blossom/fruit nodes too, not just dev mode.

### 7. GAP-C (BUGFIX-06) — First LLM response after cold start is fast
expected: First roundtrip after relaunch not ~1 min.
result: **IMPROVED, accepted (round 3, 2026-05-21).** "A little wait, but far better than before." The 55.1-07 boot pre-warm landed. Residual latency is attributed by the operator to model-provider reasoning/thinking before streaming begins. NEW (separate) request: this is fine for Ask Q&A + categorization, but hurts the lazy-loaded POST BODY (user taps into a post and waits seconds for the model to "think" before the body streams). Asks for a **separate, configurable model entry for post-body generation only**, with thinking/reasoning disabled for low streaming latency. Tracked as a new feature (NOT cold-start) — see below.

result: **PASSED (round 3, 2026-05-21).** Empty-state card no longer contradicts the feed.
