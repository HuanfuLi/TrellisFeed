## Component Inventory

### 1. ConceptProgressCard

Inline card placed between the bento grid and the InlineInfoFlow feed. Two visual states controlled by IntersectionObserver + CSS sticky.

#### Expanded State (card in-flow, not stuck)

| Property | Value |
|----------|-------|
| Position | `position: sticky; top: 0; z-index: 100` |
| Width | 100% of parent column (full bento grid width) |
| Background | `var(--card)` |
| Border radius | `var(--radius)` (1.25rem / 20px) |
| Box shadow | `var(--shadow-1)` |
| Padding | 16px |
| Height | auto (~80px content-driven) |
| Transition | `height 200ms ease, padding 200ms ease, border-radius 200ms ease, box-shadow 200ms ease` |

Layout (expanded):
```
+---------------------------------------------+
|  [BookOpen icon 20px]  Today's Concepts      |
|  2 of 5 explored                             |
|  [===========================-----] bar 8px  |
+---------------------------------------------+
```

- Icon: `BookOpen` from lucide-react, 20px, color `var(--primary-40)`
- Title: "Today's Concepts" — 16px weight 600, color `var(--foreground)`
- Subtitle: "N of M explored" — 14px weight 400, color `var(--muted-foreground)`
- Progress bar: reuse `ProgressBar` component, height 8px, color `var(--primary-40)`
- Icon and title on one row (flexbox, gap 8px, align-items center)
- Subtitle on second row (marginTop 4px)
- Progress bar on third row (marginTop 8px)

#### Compact State (stuck at top)

Triggered when the card's top edge hits the viewport top (IntersectionObserver with `threshold: 1.0` on a sentinel element placed just above the card, or `rootMargin` approach).

| Property | Value |
|----------|-------|
| Background | `var(--card)` |
| Border radius | 0 (flush with viewport edges) |
| Box shadow | `var(--shadow-2)` |
| Padding | 8px 16px |
| Height | 40px (single row) |

Layout (compact):
```
+---------------------------------------------+
| [BookOpen 16px]  2/5  [==========---] bar 6px|
+---------------------------------------------+
```

- Icon shrinks to 16px
- Counter: "N/M" — 14px weight 600, color `var(--foreground)`
- Progress bar: height 6px, flex: 1, marginLeft 12px
- Single row flexbox, align-items center, gap 8px

#### Transition Mechanism

The expanded-to-compact transition uses a CSS class toggle. A sentinel `<div>` with `height: 1px` is placed immediately above the progress card. An IntersectionObserver watches the sentinel:

- Sentinel **visible** (intersecting): card gets class `expanded` (full size)
- Sentinel **not visible** (scrolled past): card gets class `compact` (thin bar)

All animated properties use `transition: all 200ms ease`. The card MUST NOT use `position: fixed` -- it uses `position: sticky; top: 0` so it naturally pushes the greeting off-screen as the user scrolls.

#### Completion State (all concepts explored)

| Property | Change from default |
|----------|-------------------|
| Progress bar color | `#E8A838` (gold) instead of `var(--primary-40)` |
| Background | `color-mix(in srgb, #E8A838 8%, var(--card))` — subtle gold tint |
| Subtitle text | Changes to "All caught up!" (expanded) or checkmark icon (compact) |
| Confetti | `<Confetti active={true} />` — reuse existing component, fires once for 3.5s |
| Toast | "+1 earned!" via `toast()` helper |

After completion, the card stays visible at 100% gold. Feed remains fully scrollable (D-16).

### 2. Empty State (no concept posts today)

When `conceptCount === 0`, the ConceptProgressCard is NOT rendered (D-17, D-18). The feed area shows:

| Property | Value |
|----------|-------|
| Icon | `Sparkles` from lucide-react, 32px, color `var(--muted-foreground)` with 0.5 opacity |
| Heading | "Nothing new today" — 16px weight 600, color `var(--foreground)` |
| Body | "Check back later for fresh concepts to explore." — 14px weight 400, color `var(--muted-foreground)` |
| Layout | Centered vertically in a 160px min-height container, text-align center |
| Background | transparent (inherits page surface) |

### 3. PostDetailScreen Reading Indicators (no visual change)

Reading detection is invisible to the user. No progress indicators inside PostDetailScreen itself. The three triggers (scroll 70%, dwell 30s, follow-up question) emit `CONCEPT_EXPLORED` silently via event bus.

## Interaction Contracts

### Scroll Behavior

1. User opens HomeScreen. Greeting header shows at top (fixed). Below: bento grid, then ConceptProgressCard (expanded), then feed posts.
2. User scrolls down. The greeting header (`<Header>`) is fixed at `z-index: 190`. The progress card has `position: sticky; top: 0; z-index: 100`. As the user scrolls, the card sticks below the fixed header.
3. **Correction to D-08/D-11:** The greeting lives in a fixed `<Header>` component (z-index 190, position fixed). The sticky card cannot push it off-screen. Instead: the sticky card sticks at `top: calc(var(--safe-area-top) + 56px)` (below the header). In compact mode, it forms a secondary bar beneath the greeting header.
4. When the sentinel above the card scrolls out of view, the card transitions to compact state via class toggle (200ms ease).
5. Scrolling back up: sentinel re-enters view, card transitions back to expanded state.

### Reading Detection Flow

1. User taps a concept post in the feed -> navigates to `/posts/:id`.
2. PostDetailScreen mounts. Three independent detectors activate:
   - **Scroll sentinel**: IntersectionObserver on a `<div>` at 70% essay depth. Fires once.
   - **Dwell timer**: `setTimeout` at 30000ms from mount. Clears on unmount.
   - **Follow-up**: Listens for thread reply submission event.
3. First trigger to fire emits `CONCEPT_EXPLORED { anchorId }` on event bus.
4. HomeScreen subscriber increments progress count. `dailyReadService` persists to localStorage.
5. If all concepts explored: gold transition + confetti + toast + credit award.

### Daily Reset

- `dailyReadService` stores `{ date: 'YYYY-MM-DD', exploredAnchors: string[] }` in localStorage.
- On HomeScreen mount, if stored date !== today, reset to empty.

## Bento Card Decision

**Decision: DEFER the bento concept-topic card.** Per D-20, v1 caused empty-space layout issues. The current bento grid has 3 cards (Flashcard, Planner, Podcast). Adding a 4th would require either:
- A 4th card in the 2-column grid (uneven row), or
- Replacing the full-width podcast card (breaking existing layout)

Neither option is clean. The ConceptProgressCard already communicates the same information (concept count + progress) in a more contextual location (directly above the feed). Adding a bento card would be redundant. **Drop entirely for Phase 30.**

## Dark Mode Contract

All colors use CSS variables that already have dark-mode counterparts in `index.css`:

| Light | Dark | Used for |
|-------|------|----------|
| `var(--card)` #FFFFFF | `var(--card)` (dark: inherits theme) | Card background |
| `var(--primary-40)` #558B2F | `var(--primary-40)` #4CAF50 | Progress fill |
| `var(--foreground)` #2D2D2D | `var(--foreground)` (dark value) | Title text |
| `var(--muted-foreground)` #6B6B6B | `var(--muted-foreground)` #A0A0A0 | Subtitle, empty body |
| `var(--shadow-1)` | `var(--shadow-1)` (dark value) | Card shadow |
| `#E8A838` (gold) | `#E8A838` (same) | Completion — high contrast on both themes |

The gold `#E8A838` provides sufficient contrast on both `--surface` light (#FFFBF5) and dark (#111111).
