## Spacing Scale

Declared values (reuse existing `--space-*` tokens from Phase 28 D-26):

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| xs / `--space-xs` | 4px | Gap between progress label and bar track |
| sm / `--space-sm` | 8px | Internal padding within progress bar header (vertical) |
| md / `--space-md` | 12px | Bento card icon-to-text gap |
| lg / `--space-lg` | 16px | Horizontal padding of progress bar header (matches content padding) |
| 2xl / `--space-2xl` | 24px | Bento card internal padding (matches existing bento cards) |
| 3xl / `--space-3xl` | 32px | Not used this phase |

Exceptions: `--space-md: 12px` is a pre-existing Phase 28 token (D-26) already declared in `app/src/index.css`. It is reused here, not newly introduced. While 12px is not in the standard 8-point set {4, 8, 16, 24, 32, 48, 64}, it is grandfathered as an established project token.

## Component Inventory

### 1. FeedProgressHeader (new inline component in HomeScreen)

**Renders when:** `dailyPosts.length > 0`
**Hidden when:** `dailyPosts.length === 0` (D-13, D-14 -- never show 0/0)

| Property | Value |
|----------|-------|
| Position | `position: fixed` |
| Top | `HEADER_HEIGHT` px (below the standard `<Header>`) |
| Left / Right | `0` |
| z-index | `40` (below sub-screen overlays at 50) |
| Background | `var(--surface)` |
| Padding | `8px 16px` |
| Border bottom | `1px solid var(--border)` |
| Height (rendered) | ~44px (8px top + 16px label + 4px gap + 8px bar + 8px bottom) |

**Layout:**

```
+--------------------------------------------------+
| "2 of 4 today"            [ProgressBar 0-100%]   |
+--------------------------------------------------+
```

Use a flex row: label on left, ProgressBar expanding to fill remaining width. Override the default ProgressBar label rendering (which places label ABOVE the bar) by passing `label={undefined}` to `<ProgressBar>` and rendering the label text as a sibling `<span>` in the flex container.

| Sub-element | Style |
|-------------|-------|
| Label span | `fontSize: '0.75rem'`, `color: 'var(--muted-foreground)'`, `whiteSpace: 'nowrap'`, `marginRight: '12px'`, `minWidth: '80px'` |
| ProgressBar wrapper | `flex: 1`, `alignSelf: 'center'` |
| ProgressBar height | `6` (slightly thinner than default 8 for header compactness) |
| ProgressBar color (active) | `var(--primary-40)` |
| ProgressBar color (complete) | `#E8A838` |
| Transition | Inherited from ProgressBar: `width 0.4s ease` |

**Visibility guard for SwipeTabContainer:** Conditionally render only when `location.pathname === '/home'` to prevent the fixed element from bleeding into other screens (Pitfall 1 from RESEARCH.md).

**Content padding adjustment:** When progress bar is visible, increase `paddingTop` of the scrollable content area by 44px (the rendered height of the progress header).

### 2. Feed Bento Card (new card in HomeScreen bento grid)

**Position in grid:** Half-width card. Grid order: Flashcard (half) + Feed (half) on row 1, Planner (half) + empty or reflow on row 2, Podcast (full-width) on row 3. Alternative: keep Flashcard + Planner on row 1, Feed + [empty] on row 2, Podcast full-width on row 3. Use the simpler option: place Feed as the 3rd half-width cell, making Planner and Feed on row 2 (2x2 grid naturally reflows).

Grid order: Flashcard, Planner, Feed, Podcast (full-width).

| Property | Value |
|----------|-------|
| Background | `var(--bento-feed-bg)` |
| Icon | `Newspaper` from lucide-react, size 28, color `var(--bento-card-text)` |
| Title | `t('home.feed.bentoTitle')` = "Today's Feed" |
| Topic list | Up to 2 topic names from `dailyPosts[*].keywords[0]`, deduplicated |
| Overflow | `t('home.feed.bentoMore', { count: N })` = "+N more" |
| Font size (topics) | `0.875rem` (14px), color `var(--bento-card-text-muted)` |
| Tap action | Scroll the container to the `InlineInfoFlow` section using `scrollIntoView({ behavior: 'smooth' })` |
| Empty guard | Do not render when `dailyPosts.length === 0` |

### 3. Empty State (replaces feed area when no posts)

**Renders when:** `dailyPosts.length === 0`

| Property | Value |
|----------|-------|
| Container | `textAlign: 'center'`, `padding: '48px 24px'` |
| Icon | `Coffee` from lucide-react, size 48, color `var(--muted-foreground)`, `opacity: 0.5` |
| Heading | `t('home.feed.emptyTitle')` at 16px weight 600, color `var(--foreground)` |
| Body | `t('home.feed.empty')` = "Your feed is brewing -- ask a question to get started" at 14px weight 400, color `var(--muted-foreground)`, `marginTop: '8px'` |

### 4. Celebration Overlay (on quota completion)

Reuses existing `<Confetti active={showConfetti} />` component. No custom particles needed.

| Property | Value |
|----------|-------|
| Trigger | First time `readCount >= totalCount` AND `!state.quotaCompleted` in current session |
| Confetti duration | 3500ms (matches existing harvest pattern) |
| Bar color shift | Transition from `var(--primary-40)` to `#E8A838` on `quotaCompleted` |
| Toast | `toast(t('home.feed.credits', { count: 1 }), 'success')` |
| Event | `eventBus.emit({ type: 'DAILY_QUOTA_COMPLETED', payload: { creditsAwarded: 1 } })` |
| Credit award | `trellisCreditsService.add(1)` (once per day, guarded by `quotaCompleted`) |

No fly-to-counter animation from HomeScreen (cross-screen ref not available). PlannerScreen reacts to `DAILY_QUOTA_COMPLETED` event to refresh its counter.

## Interaction States

### Progress Bar States

| State | Bar Value | Color | Label |
|-------|-----------|-------|-------|
| No posts | Not rendered | n/a | n/a |
| 0 read | 0% | `var(--primary-40)` | "0 of 4 today" |
| Partial (e.g., 2/4) | 50% | `var(--primary-40)` | "2 of 4 today" |
| Complete (4/4) | 100% | `#E8A838` gold | "All caught up!" |
| Complete + reloaded | 100% | `#E8A838` gold | "All caught up!" (no re-celebration) |

### Read Detection

| Trigger | Behavior |
|---------|----------|
| Card bottom edge exits viewport top | `onPostRead(postId)` fires |
| Same card scrolled past again | Idempotent -- no state change |
| App restart same day | `dailyReadService.getState()` restores `readIds` from localStorage |
| App restart next day | `dailyReadService` auto-resets (date mismatch) |
| Mount race (Pitfall 2) | 200ms grace period OR track previously-intersecting cards before marking as read |

### Bento Card States

| State | Appearance |
|-------|------------|
| Posts exist, 0 read | Shows topic names, tappable |
| Posts exist, some read | Same as above (no visual change on bento card for read progress) |
| No posts | Bento card hidden |

## Accessibility

| Concern | Solution |
|---------|----------|
| Progress bar semantics | Add `role="progressbar"`, `aria-valuenow={readCount}`, `aria-valuemin={0}`, `aria-valuemax={totalCount}`, `aria-label={t('home.feed.progress', { read, total })}` on the ProgressBar wrapper |
| Color contrast (label on surface) | `var(--muted-foreground)` on `var(--surface)` = #6B6B6B on #FFFBF5 = 4.8:1 ratio (passes AA) |
| Color contrast (dark mode) | #A0A0A0 on #111111 = 7.4:1 ratio (passes AAA) |
| Gold bar on surface | #E8A838 on #F5F0E8 track = 2.5:1 (decorative, not text -- acceptable) |
| Empty state icon | Decorative only (`aria-hidden="true"`) |
| Bento card tap target | Entire card is a `<button>` (matches existing bento pattern) with min 44px height |
