    - Reset ALL flashcards linked to this anchor: `flashcardService.getAll().filter(c => c.nodeId === anchorId || qaChildIds.includes(c.nodeId ?? ''))` then for each call `flashcardService.updateReviewSchedule(c.id, { nextReviewDate: today(), reviewCount: 0, easeFactor: 2.5 })`

    - Generate a post for the anchor's topic: `await conceptFeedService.generateMorePosts([anchorQuestion])` — await this
    - Return navigation state: `{ navigateTo: '/review', state: { anchorReview: { anchorId, qaIds: qaChildIds, title: anchorQuestion.title ?? anchorQuestion.content } } }`
    - Per D-13: Do NOT generate new flashcards. Use existing ones.
    - Per RESEARCH Pitfall 2: MUST reset flashcards (not just anchor question) because computeLeafState uses fcMap as authoritative source

    **3. `prune(anchorId: string)`** (per D-15, D-16):
    - Call `questionService.patchQuestion(anchorId, { flagged: true })`
    - Emit `eventBus.emit({ type: 'ANCHOR_DELETED', payload: { anchorId } })` — reuse existing event (per RESEARCH Open Question 4: same trellis effect as deletion)
    - Return `{ pruned: true }`

    **4. `unpruneQuestion(anchorId: string)`** (for pruned section restore):
    - Call `questionService.patchQuestion(anchorId, { flagged: false })`
    - Emit `eventBus.emit({ type: 'CLASSIFICATION_COMPLETED', payload: { anchorId, anchorName: '' } })` to trigger trellis recompute

    **5. `hardDelete(anchorId: string)`** (for pruned section permanent delete):
    - Call `questionService.delete(anchorId)` — already emits QUESTION_DELETED
