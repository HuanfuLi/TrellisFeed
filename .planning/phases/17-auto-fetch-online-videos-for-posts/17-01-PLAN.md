Create `app/src/services/youtube.service.ts` following the ServiceResult pattern. Per D-01, D-02, D-03, D-08, D-09, D-10.

The service exports `youtubeService` with these methods:

**1. `searchVideos(query: string, maxResults?: number): Promise<ServiceResult<YouTubeSearchResult[]>>`**
- Read API key from `mockSettingsService.getSync().youtube?.apiKey`
- If no API key, return `{ success: false, error: { code: 'NO_API_KEY', message: 'YouTube API key not configured in Settings', retryable: false } }`
- Use `fetch()` (YouTube Data API v3 supports CORS) to call:
  `https://www.googleapis.com/youtube/v3/search?part=snippet&type=video&videoEmbeddable=true&q=${encodeURIComponent(query)}&maxResults=${maxResults}&relevanceLanguage=en&safeSearch=strict&key=${apiKey}`
- Parse response items into `YouTubeSearchResult[]`: `{ videoId, title, description, thumbnailUrl (use snippet.thumbnails.high.url), channelTitle }`
- Handle 403 quotaExceeded with specific error message
- Quota cost: 100 units per call (document in comment)

**2. `fetchTranscript(videoId: string): Promise<string | null>`**
- Import `{ Capacitor, CapacitorHttp }` from `@capacitor/core`
- If native (`Capacitor.isNativePlatform()`): use CapacitorHttp.get to fetch `https://www.youtube.com/watch?v=${videoId}` with header `Accept-Language: en`
- If browser: use standard `fetch()` — this will likely fail due to CORS, catch and return null
- Extract `captionTracks` from response via regex: `/"captionTracks":\s*(\[.*?\])/`
- Find English track (prefer manual `kind !== 'asr'`, then auto, then first available)
- Fetch caption XML from `track.baseUrl` (via CapacitorHttp on native, fetch in browser)
- Parse XML: strip tags, decode HTML entities (`&amp;`, `&lt;`, `&gt;`, `&#39;`, `&quot;`), collapse whitespace
- Return plain text string, or null if extraction fails at any step
- Wrap everything in try/catch returning null on failure

**3. `summarizeTranscript(transcript: string, videoTitle: string): Promise<ServiceResult<string>>`**
- Per D-10: use existing `chatCompletion` with `serviceName: 'video-summary'`
- Read LLM config from `mockSettingsService.getSync().llm`
- Truncate transcript to first 4000 chars to avoid token limits
- System prompt: "You are an educational content summarizer. Given a YouTube video transcript, produce a clear, concise summary (200-400 words) that captures the key educational concepts, examples, and takeaways. Use markdown formatting."
- User message: `Video title: "${videoTitle}"\n\nTranscript:\n${truncated}`
- If transcript is null/empty, fall back to title+description summary: system prompt asks for summary from title only (mark as description-based)

**4. `generateVideoPosts(count?: number): Promise<DailyPost[]>`**
- Per D-01: Get due review items via `mockReviewService.getTodayReviewItems()`
- Extract concept names: prefer `title`, fall back to `content.slice(0, 60)`, filter out anchor/cluster nodes
- Per research pitfall 1: Group concepts into ceil(count/2) search queries (combine 2-3 related concepts per query) to conserve quota
- Call `searchVideos()` for each query with maxResults = ceil(count / numQueries)
- Deduplicate by videoId
- For each result (up to `count`, default 3 per D-03):
  - Fetch transcript
  - If transcript available, summarize it
  - If no transcript, summarize from title+description (mark summary as description-based)
  - Build a DailyPost with:
    - `id`: `video-${videoId}`
    - `date`: `today()` from lib/date
    - `title`: YouTube video title
    - `teaser`: `{ hook: channelTitle, preview: description.slice(0, 120) }`
    - `bodyMarkdown`: the AI summary (or "Summary unavailable" fallback)
    - `whyCare`, `takeaway`, `quickAskPrompts`: empty/placeholder values (video detail shows summary differently)
    - `narrativeMode`: `'mechanism-breakdown'`
    - `contextLabel`: `'Video from YouTube'`
    - `sourceType`: `'video'`
    - `sourceQuestionIds`: IDs of the due questions that drove the search
    - `sourceQuestionTitles`: titles of those questions
    - `keywords`: from video title words
    - `generatedAt`: `Date.now()`
    - `origin`: `'ai'`
    - `videoMeta`: `{ videoId, channelTitle, thumbnailUrl, transcript, summary, duration }`
- Return the array of DailyPost objects

**5. Video post cache**: Store generated video posts in localStorage key `echolearn_video_cache` with `{ date, posts }` structure. On `generateVideoPosts`, check cache first — if date matches today and posts exist, return cached. On `generateMoreVideoPosts(count = 4)`, append to cache.

**6. `generateMoreVideoPosts(count?: number): Promise<DailyPost[]>`**
- Per D-03: default count = 4
- Same logic as generateVideoPosts but skips already-cached videoIds
- Appends to cache

**7. `getCachedVideoPosts(): DailyPost[]`**
- Read `echolearn_video_cache` from localStorage
- Parse JSON, check if `parsed.date === today()`
- If today's date matches, return `parsed.posts ?? []`
- If stale or missing, return `[]`
- Wrap in try/catch, return `[]` on any error
- This method is the ONLY way other services should read the video cache (avoids raw localStorage access elsewhere)

Add 1-2 second delay between transcript fetch calls to avoid rate limiting (per research open question 1).
