<objective>
Close the major gap from Phase 49 UAT Test 1: long-press menu must fire at the 480ms timer tick (matching feed-tile `useLongPress.ts:42-45` convention) AND long-press-drag must suppress MindElixir's pan/zoom so the dragged node can be repositioned relative to other anchors.

Two root causes diagnosed in `.planning/debug/phase-49-gesture-engine.md`:

**Finding 1 â€” wrong state-machine shape.** `useLongPressOrDrag.ts:104-108` only sets internal `didLongPress=true` + fires haptic inside the 480ms timer. No consumer-visible callback fires until `onPointerUp` runs. GraphScreen's CorrectionCard mount is wired to `onLongPressRelease` (lines 210, 248, 268, 274, 439, 769, 1256), so the card only appears AFTER the user lifts their finger â€” diverging from the feed-tile convention.

**Finding 2 â€” MindElixir co-registered listeners not suppressed.** MindElixir registers its own pointerdown/move/up listeners on the SAME container GraphScreen attaches its delegated listener to (`MindElixir.js:1095-1101`). With `editable: false`, MindElixir's pointerdown ALWAYS falls into the pan branch (`MindElixir.js:1044-1045`: `t.mousedown = true` + `setPointerCapture`). Every subsequent pointermove pans the map. GraphScreen never calls `mei.dragMoveHelper.clear()`, never attaches a capture-phase `stopPropagation()` listener, never overrides MindElixir's pointer capture. The pre-existing `setPointerCapture` at GraphScreen.tsx:446 fires only inside `onDragStart` (480ms + 8px later) â€” too late and on the wrong element (the target node, not the container).

Purpose: flip `nyquist_compliant: false â†’ true` in `49-VALIDATION.md` after on-device UAT re-passes Tests 1, 2, 3 on a real Capacitor Android/iOS build.

Output: rewritten hook + factory with the new `onLongPressRecognized` callback; rewired GraphScreen with 3 layered defenses against MindElixir pan; correctionNode-drag-clear (D-01 invariant); updated load-bearing comment; new integration test for cross-library coordination; rewritten unit tests; on-device sign-off; flipped VALIDATION.md.
