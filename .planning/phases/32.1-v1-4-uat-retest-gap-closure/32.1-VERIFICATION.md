### Architectural decisions (durable — encoded in CLAUDE.md / inline comments)

- **D-W4-01 (Cycle-stamped post IDs):** Any deterministic ID schema that gets re-generated across cycles MUST include the cycle number to avoid silent dedup-induced data loss. `seenPostIds` in `infiniteScrollService` survives across dequeue calls but `postQueueService.enqueue`'s within-queue dedup doesn't see it — so deterministic IDs that re-enter the queue get filtered AFTER appearing fresh. Pattern: append `${cycleNumber}` (or timestamp) to any post-id template that depends only on date + concept.
- **D-W4-02 (Style-assignment caller capability check):** `assignStyles` returns ANY style from the global pool, but each caller has a different generation capability. Callers MUST filter the assignment to styles they can actually fulfill. Session-post path can only emit text content (no YouTube/Tavily fetch) → must restrict to text-only. Future callers with similar partial capability MUST do the same check.
- **D-W4-03 (Render-layer defense net):** `InfoFlow.tsx`'s `wouldRenderVisual` check is the LAST line of defense before a card renders empty. Pattern: if NO visual block would render, force `effectivePresentationStyle = 'text-art'`. This catches future regressions in any generation path (image gen failure, missing videoMeta, missing newsMeta, missing suggestion topics, unknown `presentationStyle`, legacy `image-less`). Dev-mode `console.warn` makes regressions visible without users seeing broken cards.
- **D-W4-04 (Persisted dedup):** YouTube videoId dedup is now derived from `echolearn_post_history` (already persisted). Original D-02 choice ("in-memory only; fresh session can re-show") reversed — operator UX cost of repeated videos outweighs the rate-limit-recovery benefit. Per-day cutoff preserved (yesterday's IDs don't block today). Pattern carries to other dedup sets: prefer derivation from a persistent source over standalone in-memory.
- **D-W4-05 (Query variation for deterministic search APIs):** Identical YouTube queries return identical top-N results. Cycle-keyed query modifier rotation (`buildYoutubeQuery`) gives fresh result sets across cycles even with persistent dedup. Pattern: when calling a deterministic external search, vary the query by a cycle/seed value derivable from the caller context. Pre-validation and actual fetch MUST use the same query.
- **D-W4-06 (Vine progress fills container):** SVG with fixed `viewBox` + `preserveAspectRatio="xMidYMid meet"` inside a flex child centers content with empty side margins. Two-part fix: measure wrapper via `ResizeObserver` AND make the path geometry independent of decorator count (here: number of concept flowers). Pattern: a "progress" component should fill its container even when there are few items to display along it.
- **D-W4-07 (No lazy hardcoded names from LLM pipelines):** Multi-step LLM pipelines that short-circuit with placeholder names ("X fundamentals", `question.title`) produce ugly user-visible labels. Trade one extra LLM call for proper names; keep the conversation chain so KV cache amortizes the marginal cost. Reject the placeholder list defensively in the parser so even a disobedient LLM can't ship them.
- **D-W4-08 (Card visual consistency):** All feed cards (regular concept post, news post, suggestion card) MUST share the same shadow tier (`var(--shadow-2)`) and border treatment so they read as a unified surface in the feed. SuggestionCard previously used `var(--shadow-1)` and stood out as a separate component class.
