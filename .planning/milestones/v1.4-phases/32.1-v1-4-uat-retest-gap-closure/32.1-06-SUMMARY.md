## Outcome

AI-text-post provenance (sourceQuestionIds, sourceQuestionTitles, keywords) now derived from the style assignment, mirroring video/short/news pattern. Eliminates Bug 3 (concept badges missing on every AI text post when LLM omits/mangles sourceQuestionIds — frequent on OpenAI + fresh-install devices).

## Bug 3 root cause (operator-reported, HuanfuLi 2026-04-19)

Operator: "The concept badge at the bottom of each post is missing. In my test, none of them ever appeared." Worked previously on web with LM Studio; failed on device with OpenAI.

Root cause was NOT OpenAI-vs-LM-Studio compliance (OpenAI is the better model). It was the architecture:

- AI-text path at `concept-feed.service.ts:722-727` trusted the LLM to echo `sourceQuestionIds` back in JSON. Parser at line 431-433 filtered via `Array.isArray + typeof string + byId.has()`; on any non-compliance the field silently emptied.
- LM Studio (web) happened to echo IDs back from the rich accumulated dev-context.
- OpenAI (device, fresh install via Clear All Data) had thin context. Followed the prompt's "Use only sourceQuestionIds from the provided context" clause conservatively → emitted `[]`.
- Video/short/news at lines 807/843/876 already did the right thing — hardcode from `[a.conceptId]`.

## Fix

Single mutation block added at `concept-feed.service.ts:728-748` (after the existing presentationStyle apply, before the suggestion handling). Overrides parsed.posts[i].sourceQuestionIds/Titles/keywords with values derived from `byId.get(assignment.conceptId)`. Anchor name resolved via `concept.parentId → anchor.title` with `titleFor(concept)` fallback.

22 lines added. Type-check clean (0 errors).

## Decisions

- **L1 only** — provenance derivation. L2 (schema-validated parsing) and L3 (provider-native structured output) deferred per chat discussion: too complicated for what should be a simple fix, can land later if needed.
- **Loss accepted** — AI text posts can no longer credit multiple source anchors. The badge row only ever rendered up to 2 entries anyway, and the LLM-provided extras were noise.
- **Session-post path NOT touched** — `concept-feed.service.ts:1317-1336` has a different ordering (assignments derived FROM LLM-returned IDs). Out of L1's surgical scope; would need separate analysis.
