Close the three critical Phase-1 safety gaps at one coherent trust boundary: make affirmative consent mandatory before routes or logging, replace public numeric-account resolution/ingest with authenticated one-install enrollment, and make the real Capacitor/WebView cross-origin path usable through exact-origin CORS. Harden the existing CSV exporter before the updated Worker is deployed.

Purpose: the collection loop must be ethically gated, callable from the signed app, and resistant to assignment enumeration and forged/cross-account study records.
Output: consent policy and defense-in-depth logging guard; versioned wire contract; hashed/revocable install-token D1 migration; authenticated enrollment/ingest; CORS/preflight tests; hardened CSV; migrated live deployment with non-secret smoke evidence.
</objective>

<threat_model>
## ASVS L1 Security Enforcement

**Policy:** OWASP ASVS L1 baseline; `block_on: high`. Any unresolved high-severity item below blocks plan completion and deployment.

### Assets
- Affirmative consent state and the guarantee of zero pre-consent collection.
- Hidden account-to-condition/topic assignments.
- Enrollment credential, per-install bearer tokens, and researcher admin password.
- Integrity/ownership of behavioral and question/answer records.
- Participant-authored CSV text opened by researchers.

### Trust boundaries
- Unconsented participant/navigation -> participant routes and local research stores.
- Researcher-led device setup -> public Worker enrollment endpoint.
- Participant WebView origin -> CORS preflight -> authenticated Worker endpoints.
- Raw bearer token -> SHA-256 lookup -> D1 account ownership.
- D1 participant text -> researcher CSV/spreadsheet.

### Threats, mitigations, verification
| ID | ASVS | Threat | Severity | Mitigation | Verification |
|---|---|---|---|---|---|
| P11-T1 | V4 Access Control / V8 Data Protection | Direct routes or lifecycle calls collect before consent | high | One affirmative predicate in router, session, and final logger write boundary; remove skip | route matrix + dbQuery zero-row test |
| P11-T2 | V2 Authentication | Numeric IDs enumerate condition assignments | high | Separate enrollment credential, generic failures, no unauthenticated resolution | missing/wrong credential tests |
| P11-T3 | V3 Session / V6 Crypto | Raw tokens stored server-side or reusable across accounts | high | 256-bit random token, SHA-256 server storage, one account binding, rotation/revocation | migration inspection + token tests |
| P11-T4 | V4 Access Control | Forged body identity or cross-account ID collision overwrites data | high | derive identity from token; reject identity fields and owner mismatch before ACK | collision/revision negative tests |
| P11-T5 | V4/V5 | Wildcard CORS mistaken for authorization | high | exact allowlist plus independent endpoint auth | preflight/auth matrix and browser proof |
| P11-T6 | V5 Validation | whitespace/control-prefixed spreadsheet formula executes | high | detect normalized dangerous prefix; apostrophe entire raw cell; quote safely | export regression cases |
| P11-T7 | V8 Data Protection | credentials/URLs leak through repo or evidence | high | placeholders only, secret scan, sanitized evidence schema | `rg` classified scan + human evidence review |
</threat_model>
