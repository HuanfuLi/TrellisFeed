<objective>
Turn the current one-batch best-effort uploader into a durable self-draining outbox with deterministic crash reconciliation, permanent-row quarantine, shared versioned validation limits, authenticated identity-free wire requests, and production lifecycle registration.

Purpose: every consented record must eventually be acknowledged or remain visibly recoverable; one bad row, a process crash, or a large offline backlog must never strand later valid study data.
Output: versioned client contract adapter; self-draining authenticated queue; quarantine/recovery state; delivery receipts and reconciliation; boot wiring; fault-injection and backlog tests.
</objective>

<threat_model>
## ASVS L1 Security Enforcement

**Policy:** OWASP ASVS L1 baseline; `block_on: high`. Any unresolved high-severity durability, authorization, or data-disclosure threat blocks completion.

### Assets
- Complete local event/Q&A record set and its exact delivery status.
- Per-install bearer token.
- Correct revision ownership/idempotency for Q/A updates.
- Researcher-visible pending/recovery diagnostics.

### Trust boundaries
- Logger durable record -> separate local outbox operation.
- Persisted/corrupt outbox -> client validator -> authenticated network request.
- Server ACK -> delivery ledger -> exact-envelope deletion.
- Browser online/native resume -> reconciliation and retry lifecycle.

### Threats, mitigations, verification
| ID | ASVS | Threat | Severity | Mitigation | Verification |
|---|---|---|---|---|---|
| P12-T1 | V8 Data Protection | crash leaves durable record outside outbox | high | revision-aware delivery ledger + deterministic boot/resume reconciliation | injected post-record/pre-enqueue failure |
| P12-T2 | V8 Data Protection | delete or mark delivered before trustworthy ACK | high | exact ACK intersection; receipt-before-delete ordering; retain on receipt failure | crash/failed-write tests |
| P12-T3 | V5 Validation | poison/oversize row blocks all later records | high | shared limits, quarantine, recursive singleton isolation | head poison + later progress tests |
| P12-T4 | V2/V4 | bearer leaks in body/export/log | high | Authorization header only; identity-free body; explicit export/privacy guards | serialized-body/export negative assertions |
| P12-T5 | V7 Error Handling | transient/auth failure becomes spin loop or mass quarantine | high | classify stop vs permanent singleton; no-progress break | 401/403/5xx/zero-ACK tests |
| P12-T6 | V8 | enqueue during flight is lost | high | dirty generation + durable reread before promise clears | deferred-fetch concurrency test |
</threat_model>
