## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client→/v1/ingest | Authenticated but untrusted device submits recommendation rows |
| researcher→/admin/export.zip | Basic-auth admin boundary releasing aggregate study data |
| CSV→spreadsheet | Exported cells opened in spreadsheet tools (formula execution surface) |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-04-05 | Tampering | bindRecommendationInsert identity columns | high | mitigate | user_id/condition/topic_id bound exclusively from requireInstallAuth account (Task 1 test asserts stored identity ≠ wire) |
| T-04-06 | Tampering | cross-account recommendation ID collision | high | mitigate | Pre-insert owner lookup → 409 without overwrite or ACK (Task 1) |
| T-04-07 | Tampering | CSV formula injection via reason_text/question text | medium | mitigate | Reuse escapeCsvCell for all new columns; explicit formula-prefix test (Task 2) |
| T-04-08 | Information disclosure | export columns | high | mitigate | RECOMMENDATION_COLUMNS/PARTICIPANT_COLUMNS are closed lists containing only study IDs, ranking output, trace IDs, and timing; no device/location/contact/payload column exists |
| T-04-09 | Elevation of privilege | /admin routes | high | mitigate | No auth change: requireAdminAuth + no-store retained on both admin paths (regression covered by existing admin-auth tests) |
| T-04-10 | Repudiation | ACK of discarded rows | medium | mitigate | Non-unique analysis index only; INSERT OR IGNORE can never hit an unrelated uniqueness rule (index shape locked in 04-01) |
</threat_model>
