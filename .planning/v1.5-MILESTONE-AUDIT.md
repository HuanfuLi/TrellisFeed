## Critical Gap

### CONTENT-03: Queued News Posts Can Still Use One-Source Grounding

**Requirement:** Essay prompts include 2-3 Tavily snippets instead of `sources[0].snippet` only.

**Finding:** The direct news generation path is correct, but the active refill prefetch path is only partially wired.

Evidence:

- `app/src/services/post-essay.service.ts` consumes up to three sources through `sources.slice(0, 3)`.
- `app/src/services/concept-feed.service.ts` direct news generation sets `topSources = filtered.slice(0, 3)` when there is no prefetch cache.
- `app/src/services/concept-feed.service.ts` prefetch path stores only `const chosen = filtered[0]` in `preFetched.news`.
- `generatePostBatch(..., preFetched)` then treats the cached item as a single-source list with `topSources = [cached]`.

**Impact:** Normal queued news posts can still produce `newsMeta.sources` with one Tavily source, so the milestone has not fully met CONTENT-03 on the primary queued-feed path.

**Required closure:** Make `PreFetchCache.news` carry the filtered top 2-3 Tavily results for each concept, then store all of them into `newsMeta.sources`. Add a regression test proving the prefetch path preserves multiple sources.
