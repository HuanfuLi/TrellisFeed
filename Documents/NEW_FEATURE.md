# EchoLearn Feature Specification: Graph Navigation & Immersive Review (NEW_FEATURE.md)

## 1. Overview
This document outlines the technical specifications for a major UI/UX upgrade to the EchoLearn React application. The goal is to introduce immersive, frictionless review mechanisms and a dedicated knowledge graph management system. 

**Coding Agent Directives:**
* **High Standards:** Write production-ready, typed (TypeScript), and strictly linted code.
* **Don't Reinvent the Wheel:** Prioritize well-maintained, industry-standard community libraries (e.g., `react-native-reanimated`, `react-native-gesture-handler` for animations/swipes, and robust graph rendering libraries like `react-native-svg` or web-equivalent wrapper for D3/React Flow).
* **Performance:** The Info Flow relies on smooth vertical scrolling; Graph interactions require 60fps drag-and-drop. Optimize re-renders.
* **State Management:** Ensure optimistic UI updates for database writes (SQLite).

## 2. Global Navigation & Architecture Update

### 2.1. Bottom Navigation Bar Reorganization
The current bottom bar `[Home] [Ask] [Calendar] [Settings]` must be updated to a 5-item layout.
* **New Layout:** `[Home] [Graph] [Ask] [Calendar] [Settings]`
* **The `[Ask]` Button:** Must be positioned in the exact center. Stylistically treat this as a Floating Action Button (FAB) or a highly prominent primary action. 
* **Future-proofing:** Architect the `[Ask]` button component to accept `onLongPress` events (planned for holding to trigger TTS/Voice Input).

## 3. Home Screen Revamp

The Home Screen requires structural changes to reduce the friction of initiating a review session.

### 3.1. Renaming & Layout
* Rename the existing `[Review]` button/block to `[Flashcard]` to clearly define its rigorous testing nature.
* Retain the `[Today's Podcast]` block as an independent, passive listening utility.

### 3.2. Info Flow Integration (Replaces "Recent Questions")
* Remove the `Recent Questions` list component at the bottom of the Home screen.
* Replace it with the **Info Flow (Hybrid Stream)** viewport. 
* **UX Intent:** When the user opens the Home screen and scrolls down past the summary/buttons, they smoothly and continuously transition into a full-screen, vertical swipe-based review feed (TikTok/Reels style interaction).

## 4. Feature Details: Info Flow (Hybrid Stream)

The Info Flow is a continuous vertical FlatList/ScrollView containing two distinct types of cards. The feed must inject these cards based on the daily review queue generated by the SM-2 algorithm.

### 4.1. Type A: Concept Card (Active Recall)
* **Purpose:** To review a single node/question due for today.
* **Content:** Displays the generated summary or front/back of a flashcard. 
* **Interaction Constraint:** The vertical scroll MUST be locked (or heavily paginated) until the user interacts. 
* **Rating Mechanism:** The bottom of the card must display the 1-5 rating buttons (1=Blackout, 5=Perfect). 
* **Data Flow:** Pressing a rating triggers `ReviewService.submitReview(id, rating)`, updates the local SQLite DB, and unlocks the scroll to the next card.

### 4.2. Type B: Connection Card (Graph Exploration)
* **Purpose:** To display an edge (relationship) between Concept A and Concept B.
* **Content:** "Concept A is related to Concept B because..." (Data sourced from the `relatedQuestionIds` array or AI-generated summaries).
* **Interaction:** No rating required. Vertical scroll is free.
* **"Aha!" Action:** Implement a `DoubleTap` gesture recognizer on the card. 
* **Data Flow:** Double-tapping triggers an animation (e.g., a glowing brain or spark icon) and updates the local database to increment the weight/strength of the edge between these two nodes.

## 5. Feature Details: Graph View (`[Graph]` Tab)

This is a newly created dedicated screen accessible via the new bottom navigation tab. It manages the visualization and manual mapping of the knowledge base.

### 5.1. View State 1: The Master Map
* **Default State:** A pan-and-zoomable canvas displaying the user's mapped knowledge nodes and edges.
* **Technical Requirement:** Use a performant graph library. Nodes should represent `Questions`/`Concepts`, and edges represent `relatedQuestionIds`.

### 5.2. View State 2: Card Stack Inbox (Classification Mode)
* **Trigger:** A prominent button/overlay on the Graph screen indicating "X Unlinked Nodes" (questions asked recently that haven't been manually connected).
* **Layout:**
    * **Top (Active Node):** A swipeable card component (Card Stack) displaying the first unlinked concept.
    * **Center (Magnetic Mini-Map):** 3 to 4 existing nodes fetched via semantic similarity (using local embeddings) floating as drop targets.
    * **Bottom (New Domain Bucket):** A visual drop zone indicating "Create New Category/Domain".
    * **Action Row:** A "Refresh/Shuffle" button to re-fetch the 3-4 recommended nodes.
* **Drag & Drop Interaction:**
    1.  User drags the Active Node from the top.
    2.  Dropping it onto a floating existing node creates an edge (adds ID to `relatedQuestionIds`).
    3.  Dropping it into the "New Domain Bucket" assigns it as a root node with a new category.
* **Batch Processing:** Once dropped, the current card animates away, and the next unlinked node in the stack scales up to take its place. The center recommended nodes re-render based on the new active node.

## 6. Development Checklist & API Hooks

Agent, ensure the following service methods are utilized or implemented to support this UI:

* **Info Flow Data:** Create a new query combining `ReviewService.getTodayReviewItems()` with random related-edge queries to feed the Hybrid Stream array.
* **Graph Operations:** * `GraphService.getUnlinkedNodes(): Promise<Question[]>`
    * `GraphService.getSimilarNodes(embedding: number[], limit: number): Promise<Question[]>`
    * `GraphService.linkNodes(sourceId: string, targetId: string): Promise<void>`
* **Gesture Handling:** Strict use of React Native Gesture Handler for the double-tap, vertical paging (Info Flow), and absolute positioning drag-and-drop (Card Stack). Avoid basic React Native `PanResponder` for complex animations.
